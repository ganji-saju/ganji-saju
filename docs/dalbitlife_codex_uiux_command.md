## 0. 핵심 결론

현재 달빛인생은 **사주아이식 990원 저가 운세 레이아웃을 상당히 직접적으로 따라간 흔적**이 강하다. 홈의 상품 구조, `990원 사주`, `궁합`, `대운`, `택일`, `오늘 무료`, `상담` 구성과 문장 톤이 사주아이와 너무 가깝다.

반면 GitHub main 코드에는 이미 `달빛선생`, 다크/골드 톤, 질문형 시작, 저장/다시보기/공포 표현 없음, 고전 근거 UI, PDF/대화 연결 등 **더 차별화된 방향**이 들어가 있다. 즉, 문제는 단순 UI 개선이 아니라 다음 3가지다.

1. **배포/브랜치 싱크 문제**: 공개 사이트와 main 코드의 톤·구조가 다르다.
2. **브랜드 차별화 문제**: 사주아이와 너무 유사한 홈/상품 카드/가격 서사를 걷어내야 한다.
3. **해석 UX 문제**: 결과 문장이 여전히 길고 점잖고 반복적이다. “짧고 강한 판정 → 이유 → 행동” 구조로 재설계해야 한다.

## 2. 새 포지셔닝

### 기존 인상

> “990원 사주, 궁합, 대운, 오늘운을 눌러보는 저가 운세몰”

## 3. 홈 화면 개편 방향

### 문제 인식

현재 홈은 `사주 990원`, `궁합 990원`, `대운 990원`, `오늘 무료`, `타로 무료`, `상담 무료 시작` 식의 상품몰 구조다. 사용자의 실제 고민보다 상품 카테고리가 먼저 보인다.

### 개선 원칙

홈은 상품 진열장이 아니라 **질문 선택기**가 되어야 한다.

### 개편 IA

```text
[Hero]
오늘의 선택, 길게 묻지 말고 짧게 정리하세요.
CTA 1: 지금 고민 고르기
CTA 2: 오늘 1분 운세 보기

[Question Cards]
1. 돈이 자꾸 새나요?
2. 연락해도 될지 애매한가요?
3. 이직/계약을 밀어도 될까요?
4. 가족과 또 부딪혔나요?
5. 올해 판을 바꿔도 될까요?

[1-Minute Sample]
판정: 오늘은 밀기보다 확인.
조심: 같은 말을 반복하면 힘이 빠집니다.
액션: 중요한 연락은 한 번 적고 보내세요.
근거: 갑목 일간 + 금 기운 압박

[Trust]
저장 가능 · 다시보기 가능 · 공포성 단정 없음 · 근거 펼침

[Pricing Teaser]
무료: 핵심 한 줄
990원: 주제별 정밀 해석
3,900원+: 월별/올해 전략
멤버십: 대화 + 보관 + PDF
```

### 제거/축소할 요소

- `고양이`, `츄르`, `1냥` 등 사주아이식 화폐/캐릭터 연상 요소
- `용하다고 소문난`, `비법`, `정통` 같은 전통 운세몰 문구
- 홈 첫 화면의 과도한 카테고리 나열
- `사주용선생`, `타로토선생`, `궁합양선생`, `명리호선생`처럼 서비스마다 이름이 바뀌는 구조

## 4. 해석 문장 시스템: Punch Copy Engine

### 문제 인식

현재 결과/예시 문장은 다음 패턴이 반복된다.

- “흐름은 나쁘지 않습니다.”
- “차분히 살피면 좋습니다.”
- “부드럽게 이어집니다.”
- “기운이 올라옵니다.”
- “오늘 바로 해볼 수 있습니다.”

문장이 틀린 것은 아니지만, 사용자가 결제 후 기대하는 임팩트가 약하다.

### 새 출력 구조

모든 무료/유료 결과의 첫 화면은 아래 5개 필드로 통일한다.

```ts
export interface PunchReading {
  verdict: string;        // 18~28자. 한 줄 판정.
  why: string;            // 35~55자. 근거 요약.
  caution: string;        // 25~45자. 조심할 패턴.
  action: string;         // 25~45자. 오늘 할 행동.
  evidence: string[];     // 1~3개. 일간/오행/십신/대운/합충 등.
  tone?: 'sharp' | 'warm' | 'premium';
}
```

### 문장 규칙

| 항목 | 규칙 |
|---|---|
| verdict | 명사형/단문형. “오늘은 결정 말고 확인.”처럼 끝낸다. |
| why | 전문어 1개 + 생활어 1개를 연결한다. |
| caution | 불안 조장 금지. 행동 패턴만 지적한다. |
| action | “하세요”보다 “보내기 전 저장”, “회의 전 숫자 확인”처럼 구체 행동. |
| evidence | 접히는 근거 카드로 이동. 본문을 잡아먹지 않는다. |

### 금지어/제한어

```text
금지 또는 강한 제한:
- 대흉, 반드시, 절대, 실패, 파국, 이별 확정
- 용하다고, 소름, 비법, 신점처럼 들리는 표현
- 흐름, 기운, 차분히, 부드럽게, 좋습니다 반복
- “~하실 수 있습니다”를 모든 문장 끝에 붙이는 방식
```

### Before / After

| 주제 | Before | After |
|---|---|---|
| 오늘 | 흐름은 나쁘지 않습니다. 오늘은 작게 움직이며 반응을 보는 쪽이 가장 맞습니다. | 오늘은 밀기보다 확인. |
| 돈 | 재물 흐름은 안정적이나 지출 습관을 정리하면 좋습니다. | 돈은 버는 쪽보다 새는 쪽이 문제. |
| 연애 | 감정의 온도와 표현의 타이밍을 중심으로 읽어드립니다. | 연락은 가능. 결론부터 묻지 마세요. |
| 일 | 역할 변화와 이직 타이밍을 현실적으로 정리합니다. | 이직보다 역할 재정의가 먼저. |
| 가족 | 반복되는 말과 역할의 엇갈림을 봅니다. | 싸움의 원인은 말투보다 역할입니다. |

### 샘플 결과 카드

```text
판정
오늘은 밀기보다 확인.

왜
금 기운이 강해 말이 날카로워지기 쉽습니다.

조심
같은 말을 반복하면 설득력이 떨어집니다.

액션
중요한 연락은 한 번 적고 보내세요.

근거
甲 일간 · 금 압박 · 관계운 조율 구간
```

## 5. 결과 페이지 UX 재설계

### 상단 800px 규칙

결과 페이지의 첫 800px에는 아래만 보여준다.

1. `판정 카드`
2. `조심할 패턴`
3. `오늘 할 행동`
4. `근거 1줄`
5. `심층 리포트 / 대화 이어가기` CTA

### 새 결과 페이지 구조

```text
[Result Hero]
- Focus badge: 재물 / 연애 / 일 / 가족 / 오늘
- VerdictCard
- Caution + Action
- Evidence chips
- CTA: 심층 리포트 열기 / 달빛선생에게 이어서 묻기

[Quick Facts]
- 일간
- 오행 균형
- 강약
- 현재 대운/세운

[Topic Sections]
- 돈
- 관계
- 일
- 생활 리듬

[Classic Evidence]
- 한국어 요약 우선
- 원문은 접기
- 출처/라이선스는 작은 footer

[Premium Rail]
- 월별 타이밍
- PDF 소장
- 대화 3회
- MY 저장
```

### 점수 표시 원칙

현재는 점수 카드가 많아질 경우 사용자가 숫자만 본다. 점수는 보조 지표로만 유지한다.

```text
좋음/주의/정리/확인/보류
```

위 5개 상태 라벨을 중심으로 보여주고, 점수는 접힌 영역이나 툴팁에서만 노출한다.

## 6. 입력 플로우 개선

### 유지할 것

- 질문 먼저 선택
- 저장된 프로필/최근 입력 불러오기
- 시간 모름 옵션
- 출생지 좌표 검색
- 필수 동의 단순화

### 개선할 것

| 문제 | 개선 |
|---|---|
| 1/4인데 `3단계 빠른 입력`이라고 보여 혼선 | 실제 단계 수와 라벨 일치 |
| 생년월일/출생지 입력 설명이 많음 | 도움말은 accordion/tooltip으로 이동 |
| 첫 화면 선택지가 설명형 문장 | 고민 카드형 문장으로 변경 |
| 무료/유료 상품 연결이 뒤섞임 | 결과 후 upsell로 분리 |

### 새 입력 단계

```text
Step 1. 지금 고민 선택
Step 2. 생년월일/성별 입력
Step 3. 시간/출생지 선택
Step 4. 결과 보기
```

동의는 별도 step으로 보이기보다 결과 보기 버튼 직전에 compact consent box로 처리한다.

## 8. Codex 실행 명령: P0

아래 명령을 그대로 Codex에 전달한다.

```md
# Codex P0 Command — 달빛인생 UI/UX 차별화 및 해석 압축 개편

Repository: kionya/saju-app
Primary goal: 공개 사이트를 사주아이 모방형 990원 운세몰에서 `달빛인생 = 질문 해결형 사주 판단 보조 OS`로 전환한다.

## 1. 배포/브랜치 싱크 확인
- 현재 `https://ganji-saju.vercel.app`가 어떤 Vercel project/repo/branch/build output을 바라보는지 확인한다.
- GitHub main의 `src/app/page.tsx`와 공개 사이트 텍스트가 다른 경우, 원인을 문서화한다.
- Vercel domain alias와 production branch를 정리한다.
- 작업 결과에 `docs/deployment-sync-check.md`를 추가한다.

## 2. 사주아이 유사 카피 제거
다음 표현/구조를 public UI에서 제거 또는 전면 대체한다.
- `990원 사주`, `용하다고 소문난`, `궁합도 잘봐요`, `비법`, `정통`, `츄르/냥`류 표현
- 사주/궁합/대운/택일/오늘무료/상담 카드가 사주아이와 같은 순서·명칭·CTA로 보이는 구조
- 고양이 캐릭터가 사주아이 복냥이/츄르 구조를 연상시키는 장치

## 3. 홈 화면을 질문형으로 재구성

## 11. 짧고 강한 해석 카피 뱅크

### 오늘

- 오늘은 밀기보다 확인.
- 결정은 내일, 정리는 오늘.
- 말보다 기록이 이깁니다.
- 급한 답장일수록 한 번 늦추세요.
- 오늘은 새 일보다 마무리.

### 돈

- 돈은 버는 쪽보다 새는 쪽이 문제.
- 큰 수익보다 고정비 점검.
- 지출은 감정이 올라올 때 샙니다.
- 오늘 계약은 숫자부터 다시 보세요.
- 기회는 있지만 조건이 먼저입니다.

### 연애/관계

- 연락은 가능. 결론부터 묻지 마세요.
- 상대 마음보다 내 속도가 문제.
- 서운함은 말투보다 타이밍에서 커집니다.
- 다시 이어질 수 있지만, 압박은 금물.
- 오늘은 고백보다 온도 조절.

### 일/이직

- 이직보다 역할 재정의가 먼저.
- 제안은 좋지만 조건을 남기세요.
- 오늘은 성과보다 책임 범위 정리.
- 버틸 일과 넘길 일을 나누세요.
- 급한 결정은 상사보다 일정표에 물어보세요.

### 가족

- 싸움의 원인은 말투보다 역할입니다.
- 오늘은 설득보다 거리 조절.
- 가족 문제는 맞고 틀림보다 반복 패턴.
- 같은 말은 세 번째부터 상처가 됩니다.
- 먼저 물러나는 것이 지는 건 아닙니다.

### 올해

- 올해는 확장보다 구조 정리.
- 밀어야 할 달과 접어야 할 달을 나누세요.
- 판을 바꿀 수 있지만 순서가 중요합니다.
- 올해의 승부처는 사람보다 시스템.
- 좋은 달은 기회, 나쁜 달은 점검입니다.

## 13. 최종 운영 원칙

1. **홈은 질문을 고르게 한다.** 상품을 먼저 팔지 않는다.
2. **결과는 3초 안에 읽힌다.** 장문은 아래로 내린다.
3. **근거는 접는다.** 필요할 때만 펼친다.
4. **유료 전환은 깊이의 차이로 설득한다.** 공포/불안으로 밀지 않는다.
5. **달빛인생은 사주아이처럼 보이면 안 된다.** 가격, 카피, 카드 순서, 캐릭터 정서를 모두 재설계한다.
6. **달빛선생은 persona, 달빛인생은 product.** 브랜드 체계를 혼동하지 않는다.
7. **의료·법률·투자·위기 상황은 안전 안내 우선.** 건강운도 진단/치료 표현 금지.
