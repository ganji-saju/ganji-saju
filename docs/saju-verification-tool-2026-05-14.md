# 사주 검증 도구 사용법 (2026-05-14)

> URL: **`/admin/saju-verify`** (운영에서는 robots/sitemap 미수집 + noindex)
> PR: feature/saju-verification-tool-2026-05-14

## 1. 어디다가 붙여야 하나?

### 1단계 — 입력값 폼에 입력
브라우저에서 `/admin/saju-verify` 접속 후 폼에 다음을 입력:

| 항목 | 폼 필드 | 예시 |
|---|---|---|
| 생년월일 | `year` / `month` / `day` | 1990 / 5 / 17 |
| 출생시간 | `hour` / `minute` (모르면 비움) | 10 / 30 |
| 성별 | `gender` (남성/여성) | 남성 |
| 출생지 | `location` (텍스트) | 서울 |
| 자시 기준 | `jasi` (통합/분할) | 통합 자시 |
| 현재 기준일 | `now` (ISO, 비우면 현재) | 2026-05-14T12:00:00+09:00 |

"계산하기" 버튼 → URL 에 입력값이 저장되어 새로고침해도 결과 유지.

### 2단계 — 자동 계산 결과 확인
페이지가 다음을 자동으로 표시 (사용자 템플릿의 `[계산 결과]` 와 동일한 구조):
- 년주 · 월주 · 일주 · 시주 (간지 + 오행)
- 일간 (한자 + 오행)
- 오행 점수 (목·화·토·금·수 각각의 score/count)
- 십신 분포
- 신강/신약 + 점수
- 격국 (이름 + 카테고리 + 십신)
- 용신 · 희신(보조) · 기신
- 대운 (현재 cycle) · 세운 · 월운
- v2 풀이 검증 (점수 / 상태 / 이슈 상세)

### 3단계 — 사이트 풀이 원문 붙여넣기
"사이트 풀이 원문" 텍스트 박스에 화면에서 본 풀이 텍스트를 그대로 붙여 넣으면 즉시 자동 비교:

검사 항목 (필수):
- 년주 ganzi 가 본문에 나오는가
- 월주 ganzi 가 본문에 나오는가
- 일주 ganzi 가 본문에 나오는가
- 시주 ganzi 가 본문에 나오는가 (선택, 시간 미입력 시 생략)
- 일간 (한자 또는 오행 라벨 — 木/나무 기운 등)
- 우세 오행
- 약세 오행
- 신강/신약 (한자 또는 친근 표현 — "에너지가 강한 편" 등)
- 용신 (한자 / 친근 라벨 / 도움 기운 등)
- 대운/세운/월운 ganzi (선택)

각 항목별 결과 카드:
- **✓ 발견**: 본문에 정확히 등장
- **× 미발견**: 본문에서 못 찾음 (다른 사주 입력 풀이거나 변환 누락)

### 4단계 — 요약 배너 해석
- `필수 N/N 매칭 ✓` (jade 배경): 핵심 값이 모두 본문에 등장 → 같은 사주의 풀이로 확정
- `필수 K/N 매칭 ⚠` (amber 배경): 일부 값 미발견 → 다른 사주의 풀이거나 텍스트 변환에서 누락 가능

## 2. 사용자가 보낸 템플릿과의 매핑

사용자가 보낸 verification template:

```
[입력값]
생년월일:        ← year / month / day 폼 필드
양력/음력:       ← (현재 양력 고정. 음력 지원은 추후)
출생시간:        ← hour / minute 폼 필드 (또는 unknownTime)
성별:           ← gender 폼 필드
출생지:          ← location 폼 필드
자시 기준:       ← jasi 폼 필드
현재 기준일:     ← now 폼 필드

[계산 결과]
년주: 월주: 일주: 시주: 일간: ← 자동 표시
오행 점수: 십신 분포:        ← 자동 표시
신강/신약: 격국:              ← 자동 표시
용신/희신/기신:              ← 자동 표시
대운: 세운: 월운:             ← 자동 표시

[사이트 풀이 원문]
여기에 실제 노출되는 사주풀이 내용을 그대로 붙여넣기 ← textarea 에 붙여넣기
```

## 3. 한 줄로 다시 정리

> **"이거를 검증하려면 어디다가 붙여야해?"**
>
> → 브라우저로 `/admin/saju-verify` 접속.
> 1) 입력값 폼에 생년월일·시간·성별·출생지·자시·기준일 입력 후 "계산하기"
> 2) 자동으로 [계산 결과] 가 표시됨
> 3) textarea 에 사이트 풀이 원문 그대로 붙여넣기
> 4) 매칭 결과 카드 + 요약 배너로 같은 사주의 풀이인지 즉시 확인

## 4. 운영 노출 차단

- `robots.ts` 에 `/admin` 추가 → 검색엔진 미수집.
- `metadata.robots = { index: false, follow: false }` → 직접 URL 알아도 indexing 차단.
- 인증 게이트는 미구현(누구나 진입 가능). 운영 도구이므로 별도 인증이 필요하면 `middleware.ts` 에서 차단 권장.

## 5. 추후 개선 후보

1. 입력에 음력 옵션 추가 (현재 양력 고정).
2. 검증 결과를 JSON 으로 다운로드 (감사 보고용).
3. 여러 입력 묶음을 csv 로 일괄 검증.
4. 본문이 v2 친근 용어로 작성된 경우와 v1 한자 술어로 작성된 경우 둘 다 매칭되도록 ELEMENT_ALIASES 자동 확장 (이미 부분 지원).
5. `/admin` 전체에 IP 화이트리스트 또는 비밀번호 게이트 (`middleware`).
