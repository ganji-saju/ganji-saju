작성일: 2026-05-23 · 작성: Claude Code · 출처: `src/lib/payments/catalog.ts` 현재값 + `docs/payments/product-catalog.md §2` 일관성 audit

> 본 문서는 **제안(proposal)**. 모든 숫자는 *근거 있는 출발점*이며 최종 확정은 운영자 판단. 핵심은 개별 숫자보다 **구조**(빈 중간 티어 채우기 · 가치 기반 차등 · 결제 경로 단일화).

## 2. 발견된 구조 문제 (요약)

1. **빈 중간 구간** — 단건 최고가 3,900 ↔ 보관형 49,000 사이가 비어, 소액 단건 사용자가 보관형으로 넘어갈 중간 단계가 없음.
2. **가치/지속성 역전** — 하루짜리(오늘 자세히) 550 = 장기 콘텐츠(대운/점수 풀이 `taste_score_factor`) 550. 오래 쓰는 풀이가 단발과 같은 값.
3. **이중 결제 경로** — 동일 기능이 코인 차감 + 단건 현금 양쪽 존재 (`product-catalog.md §2.2`).
4. **멤버십 분리** — 멤버십은 AI 대화 코인 전용 → 풀이 unlock과 연결 안 됨.
5. **49,000 환불 인프라 부재**(§2.4) · 코인 만료 정책-구현 불일치(§2.3).

## 3. 제안 사다리 (작성 예정 — 다음 세션)

> 출력 중단으로 미완. 다음 세션에서 아래 원칙대로 표 완성 예정:
> - 무료(획득) → 소액 단건(단발) → **중간 묶음 상품(신설, 1만원대)** → 보관형(프리미엄) 단계 사다리
> - 콘텐츠 지속성/범위 비례 가격(단발 < 1년 < 10년 대운 < 평생)
> - 결제 경로 단일화 · 멤버십에 풀이 혜택 연결 · 보관형 가격 재검토
