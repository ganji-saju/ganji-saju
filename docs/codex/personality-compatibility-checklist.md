# Personality Compatibility Checklist

작성일: 2026-05-11
브랜치: `feature/personality-compatibility`

## 1. 라우팅

- [ ] `/compatibility`에서 성향 궁합 진입이 보인다.
- [ ] `/compatibility/input`에서 내 정보와 상대 정보를 모두 입력할 수 있다.
- [ ] `/compatibility/result`에서 결과가 정상 표시된다.
- [ ] 직접 URL 접근 시 필수 데이터가 없으면 입력 화면으로 안내된다.
- [ ] 모바일 하단 네비게이션과 상단 헤더가 겹치지 않는다.

## 2. 입력 UX

- [ ] 저장된 내 프로필을 한 번에 불러올 수 있다.
- [ ] 저장된 가족/상대 프로필을 이름만 눌러 불러올 수 있다.
- [ ] 비로그인 사용자는 내 정보와 상대 정보를 모두 입력할 수 있다.
- [ ] 생년월일은 모바일에서 화면 밖으로 밀리지 않는다.
- [ ] 연/월/일 선택 UI가 다른 사주 입력 화면과 일관된다.
- [ ] 출생시간 미입력 옵션이 있다.
- [ ] 출생지 입력값이 결과 화면까지 유지된다.
- [ ] 관계 유형 선택이 결과에 반영된다.

## 3. 무료 결과

- [ ] 첫 화면에 관계 한 줄 요약이 나온다.
- [ ] 잘 맞는 지점이 2개 이하로 짧게 나온다.
- [ ] 자주 부딪히는 지점이 2개 이하로 짧게 나온다.
- [ ] 오늘 바꿔볼 행동이 1개 나온다.
- [ ] 본문에 `격국`, `용신`, `기신`, `engine`, `evidence_json` 같은 표현이 노출되지 않는다.
- [ ] 어려운 한자 설명이 본문 흐름을 끊지 않는다.
- [ ] 두 명의 입력값을 바꾸면 결과 문구와 점수가 달라진다.

## 4. 상세 결과

- [ ] 550원 또는 1코인 상세 풀이 CTA가 명확하다.
- [ ] 결제 전에는 상세 본문이 같은 페이지 아래에 자동으로 펼쳐지지 않는다.
- [ ] 결제 완료 후 상세 결과 전용 화면으로 이동한다.
- [ ] 상세 결과는 관계 온도, 표현 방식, 갈등 패턴, 회복 문장 순서로 읽힌다.
- [ ] 각 챕터는 카드 안에 또 카드가 들어가지 않는다.
- [ ] 전문 근거는 필요 시 접힘 영역으로만 제공된다.

## 5. 결제/권한

- [ ] 상품별 `product_id`와 `scope_key`가 명확하다.
- [ ] 같은 두 사람, 같은 관계 유형, 같은 상품은 중복 결제되지 않는다.
- [ ] 1코인 해금과 550원 결제는 같은 entitlement를 공유한다.
- [ ] 이미 구매한 사용자는 결제 버튼 대신 `이미 열어본 풀이` 상태가 보인다.
- [ ] 비로그인 사용자가 결제하려 하면 로그인/회원가입 후 기존 입력값이 이어진다.
- [ ] 결제 완료 시 snapshot이 저장된다.
- [ ] 보관함에서 재열람 시 오늘 새로 생성한 결과로 바뀌지 않는다.

## 6. 보관함

- [ ] 구매한 성향 궁합이 `/my` 또는 보관함에 표시된다.
- [ ] 보관함 카드에는 관계 유형과 상대 이름이 보인다.
- [ ] 과거 결과를 열면 당시 결제한 결과가 표시된다.
- [ ] 삭제 후 목록 숫자가 즉시 반영된다.

## 7. 대화방 연결

- [ ] 결과 기반 후속 질문 CTA가 대화방으로 이동한다.
- [ ] 대화방에는 두 사람의 궁합 context가 전달된다.
- [ ] AI 답변은 질문에 직접 답한다.
- [ ] 사주 원국 설명을 길게 반복하지 않는다.
- [ ] 12간지 선생 선택이 있다면 담당 분야별 답변 톤이 달라진다.

## 8. 모바일 QA

- [ ] iPhone SE 폭에서 가로 스크롤이 없다.
- [ ] Galaxy S10급 기기에서 애니메이션이 과하게 버벅이지 않는다.
- [ ] CTA는 한 화면에 주 행동 1개, 보조 행동 1개 이하로 보인다.
- [ ] 버튼 텍스트가 줄바꿈으로 깨지지 않는다.
- [ ] 하단 dock이 마지막 CTA를 가리지 않는다.

## 9. 문구 QA

- [ ] `반드시`, `무조건`, `대흉`, `질병`, `투자 성공` 같은 단정 표현이 없다.
- [ ] `밀어도 되는`, `강하게 밀기`처럼 뜻이 애매한 표현을 쓰지 않는다.
- [ ] `정리을`, `정리이`처럼 조사 오류가 없다.
- [ ] 일반 사용자가 이해하기 어려운 명리 용어는 본문에서 줄인다.
- [ ] 결과 문장은 1~2문장 단위로 짧게 끊는다.

## 10. 검증 명령

- [ ] `npm run typecheck`
- [ ] `npm test`
- [ ] `npm run build`
- [ ] 모바일 브라우저 QA
- [ ] production 배포 후 실제 URL QA

## 11. 롤백 기준

- [ ] 기존 궁합 입력/결과 라우트가 깨지면 성향 궁합 라우트만 비활성화할 수 있다.
- [ ] 결제 entitlement 연결이 불안정하면 무료 결과까지만 노출한다.
- [ ] 보관함 snapshot 검증이 끝나기 전에는 상세 결제 버튼을 production에 열지 않는다.
