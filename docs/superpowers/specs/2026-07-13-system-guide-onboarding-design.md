# 간지사주 사용방법 온보딩 설계

## 목표

간지사주에 로그인하거나 회원가입을 마친 사용자가 전체 시스템의 핵심 사용 흐름을 처음 한 번 쉽게 이해하도록 안내한다. 사용자는 안내를 닫을 수 있고, 이후 상단 `사용방법` 메뉴에서 전체 설명서를 언제든 다시 볼 수 있다.

## 사용자 경험 원칙

- 기존 간지사주의 컬러 토큰, 폰트, 카드 반경, 버튼, 여백, 모바일 폭을 그대로 사용한다.
- 최초 안내는 한 화면에서 한 가지 행동에 집중하는 A형 단계 카드로 제공한다.
- 긴 설명 대신 2~3줄의 안내와 주 행동 하나를 보여준다.
- 사용자가 기능을 직접 이용하지 않아도 `다음`을 눌러 진행할 수 있다.
- 사용자가 `닫기`를 누르면 최초 자동 안내는 종료하며 다시 자동으로 표시하지 않는다.
- 사용방법 페이지에서는 전체 흐름을 언제든 다시 시작할 수 있다.

## 진입 방식

### 최초 자동 안내

- 로그인 또는 회원가입 완료 후 최초 1회 표시한다.
- 비로그인 방문자에게는 자동 표시하지 않는다.
- 사용자가 닫거나 마지막 단계를 완료하면 해당 브라우저에서 자동 재노출하지 않는다.
- 자동 안내를 닫은 뒤에는 상단 `사용방법` 메뉴에서만 다시 진입한다.

### 상시 메뉴

- 데스크톱 상단 주 메뉴에 `사용방법`을 독립 항목으로 표시한다.
- 같은 메뉴 항목을 모바일 주 메뉴에도 표시한다.
- 링크 목적지는 `/guide`다.

## 온보딩 단계

### 1. 내 정보 등록

- 제목: `내 정보를 먼저 등록해 보세요`
- 설명: `생년월일과 태어난 시간을 저장하면 모든 풀이가 내 정보에 맞춰집니다.`
- 주 행동: `내 정보 등록하기`
- 이동: `/my/profile`

### 2. 무료 운세

- 제목: `오늘 흐름부터 가볍게 보세요`
- 설명: `오늘운세·타로·띠운세·별자리로 오늘 흐름부터 무료로 확인하세요.`
- 주 행동: `오늘운세 보기`
- 이동: `/today-fortune`

### 3. 내 사주풀이

- 제목: `내 사주풀이를 만들어 보세요`
- 설명: `한 번 입력한 정보로 기본 풀이부터 깊은 풀이까지 이어서 볼 수 있어요.`
- 주 행동: `사주풀이 시작`
- 이동: `/saju/new`

### 4. 보관함과 공유

- 제목: `풀이를 저장하고 다시 확인하세요`
- 설명: `저장한 풀이와 결제한 콘텐츠는 보관함에서 다시 열어볼 수 있어요.`
- 주 행동: `보관함 보기`
- 이동: `/my/results`

### 5. 선생님과 대화

- 제목: `궁금한 내용을 선생님께 물어보세요`
- 설명: `주제별 선생님에게 질문하고 이전 대화도 다시 확인할 수 있어요.`
- 주 행동: `대화 시작`
- 이동: `/dialogue`

### 6. 알림과 이용 관리

- 제목: `알림과 이용내역을 한곳에서 관리하세요`
- 설명: `푸시·카카오·이메일 알림과 멤버십·결제내역을 한곳에서 관리하세요.`
- 주 행동: `알림 설정`
- 이동: `/notifications`
- 보조 행동: `멤버십 보기`
- 보조 이동: `/membership`

마지막 단계의 완료 행동은 `간지사주 시작하기`이며 홈(`/`)으로 이동한다.

## 화면 구성

### 집중형 온보딩

- 헤더: `간지사주 사용방법`, 현재 단계 `N / 6`, 닫기 버튼
- 진행 표시줄
- 단계별 아이콘, 제목, 짧은 설명
- 기능 이동용 `직접 해보기` 버튼
- `이전`, `다음` 또는 마지막 단계의 `간지사주 시작하기`
- 기능 이동 후 돌아오면 마지막으로 보던 단계를 복원한다.

### `/guide` 전체 설명서

- 페이지 제목과 짧은 소개
- 6단계 전체 진행 목록
- 각 단계의 설명과 바로가기
- `처음부터 안내 보기`로 집중형 온보딩을 수동 실행
- 자동 표시 완료 여부와 관계없이 언제든 사용할 수 있다.

## 컴포넌트 경계

- `SystemGuideContent`: 단계별 제목, 설명, 아이콘, 경로를 관리하는 단일 데이터 소스
- `SystemGuideOnboarding`: 최초 자동 표시와 수동 재실행에 사용하는 집중형 오버레이
- `SystemGuidePage`: `/guide`에서 전체 단계를 보여주는 설명서 화면
- `SystemGuideLauncher`: 로그인 상태와 브라우저 저장 상태를 확인해 자동 표시 여부를 결정
- `SiteHeader` 및 모바일 메뉴: `/guide` 링크만 담당

콘텐츠 데이터는 두 화면에서 공유하며, 설명 문구나 연결 주소가 서로 달라지지 않게 한다.

## 상태 저장

브라우저 저장소에 버전이 포함된 하나의 레코드를 저장한다.

```ts
type SystemGuideState = {
  version: 1;
  dismissed: boolean;
  completed: boolean;
  currentStep: number;
};
```

- 초기 단계는 0이다.
- `닫기`는 `dismissed: true`로 저장한다.
- 마지막 완료는 `completed: true`로 저장한다.
- 직접 해보기 이동 전 현재 단계를 저장한다.
- 저장소 접근 실패 시 화면은 계속 작동하며 상태 저장만 생략한다.
- 이번 범위에서는 DB 동기화와 여러 기기 간 완료 상태 공유를 하지 않는다.

## 표시 조건

자동 표시 조건은 다음을 모두 만족해야 한다.

- 현재 사용자가 로그인 상태다.
- 브라우저가 온보딩 상태를 읽을 수 있거나 저장 상태가 없다.
- `dismissed`와 `completed`가 모두 false다.
- 현재 경로가 로그인, 회원가입, 결제, 정책, 관리자 페이지가 아니다.

수동 실행은 표시 조건과 관계없이 `/guide`에서 가능하다.

## 접근성과 모바일 대응

- 오버레이는 body portal을 사용한다.
- `role="dialog"`, 제목 연결, 닫기 버튼의 명확한 접근성 이름을 제공한다.
- 배경 스크롤을 잠그고 오버레이 내부만 스크롤한다.
- `100dvh`, 안전영역 여백, 최대 높이를 적용한다.
- Galaxy S25와 Meta·카카오 인앱 브라우저에서도 버튼이 화면 밖으로 나가지 않게 한다.
- 키보드 포커스를 대화상자 안에 유지하고 Escape로 닫을 수 있게 한다.
- 애니메이션 감소 설정을 존중한다.

## 오류 처리

- 브라우저 저장소 오류는 사용자 흐름을 막지 않는다.
- 기능 이동 주소가 변경되지 않도록 콘텐츠 데이터의 경로를 테스트한다.
- 로그인 상태 확인이 지연되면 확인이 끝날 때까지 자동 안내를 표시하지 않는다.
- 서버 렌더링과 클라이언트 hydration 사이에 안내가 깜빡이지 않게 클라이언트 준비 후 판단한다.

## 테스트 범위

### 단위 테스트

- 기본 상태 생성과 저장 상태 정규화
- 닫기와 완료 후 자동 표시하지 않는 조건
- 마지막 단계 범위 보정
- 6단계 콘텐츠와 필수 이동 경로

### 브라우저 테스트

- 로그인한 최초 사용자에게 자동 표시
- 비로그인 사용자에게 자동 표시하지 않음
- 닫기 후 새로고침해도 재노출하지 않음
- 이전·다음 단계 이동과 복원
- 직접 해보기 이동 전 현재 단계 저장
- `/guide`에서 6단계와 수동 실행 버튼 노출
- 데스크톱·모바일 주 메뉴의 `사용방법` 링크
- Galaxy 모바일 뷰포트에서 내부 스크롤과 모든 행동 버튼 접근 가능

## 제외 범위

- 사용자별 DB 완료 상태 저장
- 기기 간 온보딩 진행률 동기화
- 각 기능의 실제 완료 여부 자동 판정
- 관리자 기능 설명
- 운영자가 문구를 수정하는 CMS

## 완료 기준

- 로그인·회원가입 후 최초 1회 안내가 표시된다.
- 닫은 사용자는 다시 자동 안내를 받지 않는다.
- 상단과 모바일 메뉴에서 `/guide`에 접근할 수 있다.
- 기존 간지사주 디자인과 모바일 동작을 유지한다.
- 확정된 6단계가 설명서와 온보딩에서 동일하게 제공된다.
- 단위 테스트, 타입체크, 빌드, 모바일 브라우저 테스트가 통과한다.
