## 1. 기능 개요

- 기능명: 달빛 성향사주
- 부제: 사주로 보는 타고난 결, 성향으로 보는 나의 선택 습관
- 목적: 개인 사주 결과에 16유형 성향 입력 또는 8문항 성향 체크를 연결해, 사용자가 자신의 타고난 기질과 선택 습관을 함께 이해하도록 돕는다.
- 핵심 질문:
  - 나는 어떤 결을 타고났을까?
  - 같은 상황에서 왜 늘 비슷한 선택을 할까?
  - 관계, 일, 돈, 올해 흐름에서 내가 쉽게 지치거나 강해지는 지점은 무엇일까?
  - 오늘 바로 바꿔볼 수 있는 작은 실행 문장은 무엇일까?

달빛 성향사주는 기존 달빛 성향궁합의 2인 관계 분석이 아니다. 기존 `src/domain/personality`의 성향 모듈은 재사용하되, 관계궁합 점수/리포트/결제 scope를 복사하지 않고 개인 사주 해석 도메인으로 분리한다.

## 2. 제품 원칙

- 사주 계산은 기존 개인 사주 엔진과 리딩 저장 구조를 그대로 사용한다.
- 성향 입력은 공식 검사나 진단처럼 보이지 않게, 참고용 자기이해 콘텐츠로 안내한다.
- 결과는 단정이 아니라 가능성, 경향, 실행 힌트 중심으로 작성한다.
- 건강/질병, 투자 판단, 직업 확정, 결혼/이별/재회 확정 표현을 제공하지 않는다.
- 기존 달빛 성향궁합의 2인 관계 로직, 관계 점수 축, 결제 상품, 저장 테이블은 변경하지 않는다.
- 무료 결과는 짧고 선명하게, 990원 깊이보기는 구체적인 해석과 실천 문장 중심으로 구성한다.

| analytics 이벤트 | 포함 | 입력, 결과, 결제, 공유, AI 상담 퍼널을 측정한다. | 이벤트 이름 registry 추가 필요 |

## 4. MVP 제외 범위

MVP에서는 아래 항목을 제외한다.

- 공식 MBTI 검사처럼 보이는 UI
- 30문항 이상 정밀 검사
- 건강/질병 예측
- 투자 판단
- 직업 확정 단정
- 성향궁합 2인 관계 로직 변경
- 기존 사주 계산 로직 변경
- 기존 성향궁합 결제 상품 재사용
- 상대방 정보 입력
- 두 사람의 관계 점수 산출
- PDF 생성
- 관리자용 리포트 편집기

주의: 금지 표현은 문서의 제외 범위 설명에서만 메타적으로 다루며, 사용자 화면/결과 본문/상품 카피에는 노출하지 않는다.

## 5. 사용자 플로우

### 5.1 진입

1. 사용자가 사주 결과 페이지, 홈 CTA, 또는 보관함에서 `달빛 성향사주`로 진입한다.
2. 기존 리딩이 있으면 `/saju/[slug]/personality`로 이동한다.
3. 기존 리딩이 없으면 `/saju/new`에서 새 생년월일시를 입력하고 리딩 생성 후 성향사주로 이어진다.
4. 로그인 사용자는 저장된 내 프로필 또는 기존 사주 리딩을 선택할 수 있다.

### 5.2 성향 입력

1. 사용자가 16유형 성향을 알고 있으면 직접 선택한다.
2. 잘 모르면 `8문항 성향 체크`를 선택한다.
3. 8문항 체크는 정확한 진단이 아니라 성향 축을 가볍게 추정하는 보조 입력으로 안내한다.
4. 성향 입력 후 관심영역을 선택한다.

### 5.3 관심영역 선택

MVP 관심영역:

- 기본 성향
- 연애
- 인간관계
- 일
- 돈/성취
- 올해
- 오늘

MVP 기본안은 단일 선택이다. 복수 선택은 결과 생성 비용과 리포트 길이가 늘어나므로 후속 확장으로 둔다.

### 5.4 무료 결과

1. 결과 페이지에서 한 줄 정의를 먼저 보여준다.
2. 6축 성향사주 점수를 요약 카드로 보여준다.
3. 무료 해석은 3문단 이내로 제한한다.
4. 유료 잠금 섹션 목록을 보여준다.
5. `990원 깊이보기`, `AI에게 이어서 물어보기`, `공유하기` CTA를 배치한다.

### 5.5 990원 깊이보기

1. 사용자가 `990원 깊이보기`를 누른다.
2. 기존 결제 준비 API 흐름으로 중복 구매 여부를 확인한다.
3. 구매 이력이 있으면 결제 없이 유료 리포트를 보여준다.
4. 구매 이력이 없으면 Toss 결제로 이동한다.
5. 결제 성공 후 유료 리포트를 저장하고 결과 화면으로 복귀한다.
6. 결제 실패 또는 취소 시 무료 결과로 복귀한다.

### 5.6 저장, 공유, AI 상담

1. 로그인 사용자는 무료/유료 결과를 저장할 수 있다.
2. 유료 결과는 결제 당시 스냅샷 또는 별도 리포트 저장소에 보관한다.
3. 공유 카드는 이름, 생년월일, 성별, 출생시간을 노출하지 않는다.
4. AI 상담 CTA는 개인 원본 정보가 아니라 요약 context만 전달한다.

## 6. 입력 데이터

### 6.1 사주 입력 데이터

기존 개인 사주 입력 구조를 재사용한다.

- 이름 또는 별명
- 생년월일
- 양력/음력
- 태어난 시간
- 태어난 시간 모름
- 성별
- 출생지 또는 시간 보정 정보

### 6.2 성향 입력 데이터

- 16유형 성향 코드
- 성향 입력 방식: 직접 선택 또는 8문항 체크
- 8문항 체크 응답
- 추정 성향 축 점수
- 관심영역

저장 원칙:

- analytics payload에는 원본 개인정보와 체크 응답 원문을 넣지 않는다.
- 공유 카드에는 개인 식별 정보와 생년월일시를 넣지 않는다.
- 리포트 저장 시 사주 원본 입력을 중복 저장하기보다 기존 `reading_id` 또는 `reading_key`와 연결한다.

## 7. 6축 성향사주 점수

MVP에서는 개인 해석에 맞는 6축을 사용한다. 점수는 모두 0~100으로 normalize한다.

| 축 | 설명 | 주요 입력 |
| --- | --- | --- |
| temperamentScore | 타고난 결과 기본 기질의 선명도 | 사주 facts 중심 |
| choicePatternScore | 반복되는 선택 습관과 판단 방식 | 성향 facts 중심 |
| expressionScore | 감정/생각을 밖으로 드러내는 방식 | 사주 facts + 성향 facts |
| relationshipStyleScore | 관계에서 편해지는 거리와 소통 방식 | 성향 facts + 사주 facts |
| workAchievementScore | 일, 성취, 돈을 다루는 추진 방식 | 사주 facts + 관심영역 |
| timingActionScore | 올해/오늘 흐름에서 실행으로 옮기기 쉬운 정도 | 사주 timing facts + 관심영역 |

주의:

- 이 점수는 운명 확정값이 아니라 자기이해를 돕는 해석용 지표다.
- 성향 유형과 오행을 1:1로 단순 매핑하지 않는다.
- 기존 성향궁합의 5축 관계 점수를 재사용하지 않는다.

## 8. 결과 리포트 구조

결과 리포트는 아래 10개 섹션을 기준으로 한다.

1. 한 줄 정의
2. 사주로 보는 타고난 결
3. 성향으로 드러나는 방식
4. 나의 강점
5. 반복해서 지치는 지점
6. 관계에서의 나
7. 일하는 방식
8. 돈과 성취 방식
9. 지금 시기의 성장 전략
10. 오늘의 실행 문장

### 8.1 무료 결과

- 한 줄 정의
- 6축 점수 요약
- 관심영역 기반 무료 해석 3문단
- 오늘의 실행 문장 1개
- 잠금 섹션 미리보기
- CTA

무료 결과는 너무 길지 않게 유지한다. 사용자가 `내 얘기 같다`는 감각을 얻되, 깊은 원인/전략은 유료 영역으로 남긴다.

### 8.2 990원 깊이보기

- 10개 리포트 섹션 전체 제공
- 반복해서 지치는 지점의 원인 해석
- 관심영역별 행동 전략
- 오늘 바로 쓸 수 있는 실행 문장
- AI 상담으로 이어가기 위한 요약 context

유료 리포트에서도 건강, 법률, 투자, 직업 확정 단정은 제공하지 않는다.

## 10. 결제/권한 정책

MVP 권장안:

1. 무료 결과는 결제 없이 표시한다.
2. 990원 깊이보기는 별도 상품으로 둔다.
3. 기존 성향궁합 상품 `personality_compatibility_mini`는 사용하지 않는다.
4. 구매 이력이 있으면 같은 scope에서 재결제 없이 유료 결과를 보여준다.
5. 프리미엄 멤버십 포함 여부는 별도 정책으로 결정한다.

신규 상품 예시:

```text
product_code: saju_personality_mini
price: 990
name: 달빛 성향사주 깊이보기
```

권장 scope 예시:

```text
saju-personality:{readingIdOrSlug}:{personalityFingerprint}:{interest}
```

scope에는 이름, 생년월일, 성별, 출생시간을 직접 넣지 않는다.

## 11. 저장/재조회 정책

MVP 권장안:

- 기존 `readings`는 개인 사주 원본 리딩으로 유지한다.
- 성향사주 결과는 기존 성향궁합 저장소와 분리한다.
- 신규 저장소가 필요하다면 `saju_personality_reports` 같은 별도 테이블을 사용한다.
- 유료 결과는 결제 당시 내용을 재조회할 수 있도록 snapshot을 남긴다.
- 비로그인 사용자는 sessionStorage 기반 임시 결과를 허용하되, 결제/저장 전환 시 로그인 요구가 필요하다.

저장 후보 데이터:

- `user_id`
- `reading_id`
- `scope_key`
- `personality_type`
- `interest`
- `score_json`
- `report_json`
- `product_code`
- `created_at`
- `updated_at`

## 12. 공유 카드

공유 카드 내용:

- 성향사주 키워드
- 6축 중 대표 키워드 2~3개
- 오늘의 한마디
- 달빛인생 로고 또는 텍스트
- 다시 보기 링크

공유 카드 금지 정보:

- 이름
- 생년월일
- 성별
- 출생시간
- 출생지
- 성향 체크 원문 응답
- 결제 여부

MVP에서는 실제 이미지 저장이나 카카오톡 공유가 준비되어 있지 않다면, 공유 데이터 구조와 UI만 먼저 제공한다.

## 13. AI 상담 CTA

AI 상담 CTA 목적:

- 사용자가 결과를 읽고 바로 이어서 질문할 수 있게 한다.
- AI 상담에는 원본 개인정보가 아니라 요약된 결과 context만 전달한다.

전달 가능 context:

- 관심영역
- 성향 유형 또는 성향 축 요약
- 6축 점수 band
- 한 줄 정의
- 오늘의 실행 문장

전달 금지 context:

- 이름
- 생년월일
- 출생시간
- 성별
- 출생지
- 성향 체크 원문 응답 전체

## 15. QA 기준

### 입력/플로우

- 기존 사주 리딩에서 성향사주로 진입할 수 있다.
- 기존 프로필 또는 리딩을 선택해 시작할 수 있다.
- 새 생년월일시 입력 후 성향사주로 이어질 수 있다.
- 16유형 직접 선택이 가능하다.
- 8문항 체크로 성향을 추정할 수 있다.
- 관심영역 7종 중 하나를 선택할 수 있다.
- 성향궁합 2인 관계 입력 화면으로 잘못 이동하지 않는다.

### 결과

- 무료 결과에 한 줄 정의와 6축 점수가 표시된다.
- 결과 리포트가 개인 사주 해석으로 보이며 2인 관계 궁합처럼 보이지 않는다.
- 입력 성향 또는 관심영역을 바꾸면 결과 요약이 달라진다.
- 무료 결과와 유료 결과의 경계가 명확하다.
- 990원 결제 전 유료 상세가 노출되지 않는다.
- 구매 이력이 있으면 재결제 없이 유료 결과를 볼 수 있다.

### 개인정보/표현

- 공유 카드에 이름, 생년월일, 성별, 출생시간, 출생지가 노출되지 않는다.
- analytics payload에 개인정보와 체크 응답 원문이 들어가지 않는다.
- 사용자 화면에 공식 검사/진단처럼 보이는 표현이 없다.
- 건강/질병 예측, 투자 판단, 직업 확정 단정이 없다.
- 결혼/이별/재회 확정 표현이 없다.
- 단정 표현은 `가능성이 있습니다`, `경향이 있습니다`, `도움이 됩니다` 같은 완화 표현으로 바뀐다.

### 회귀

- 기존 `/saju/new` 입력이 정상 동작한다.
- 기존 `/saju/[slug]` 결과가 정상 동작한다.
- 기존 `/saju/[slug]/premium` 결제/권한 흐름이 깨지지 않는다.
- 기존 `/compatibility/personality` 성향궁합 입력/결과가 변경되지 않는다.
- 기존 `personality_compatibility_mini` 결제가 변경되지 않는다.
