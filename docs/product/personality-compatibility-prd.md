## 1. 기능 개요

- 기능명: 달빛 성향궁합
- 부제: 사주로 보는 타고난 결, 성향으로 보는 관계 습관
- 목적: 두 사람의 사주 기반 관계 흐름과 16유형 성향 기반 관계 습관을 함께 보여준다.
- 핵심 질문:
  - 우리는 왜 자주 부딪힐까?
  - 상대는 어떤 방식으로 편해질까?
  - 내가 조심해야 할 말투나 행동은 무엇일까?
  - 오늘 관계를 좋게 만들려면 무엇부터 바꾸면 될까?

## 2. 제품 원칙

- 사주 계산은 기존 엔진을 그대로 사용한다.
- 16유형 성향은 공식 검사나 진단처럼 보이지 않게, 가벼운 자기이해 보조값으로만 사용한다.
- 결과는 "맞다/틀리다"가 아니라 "관계 습관을 이해하고 조정하는 힌트"로 표현한다.
- 결혼, 이별, 재회, 파국 같은 단정형 결론을 제공하지 않는다.
- 무료 결과는 짧고 바로 이해되게, 유료 결과는 실천 문장 중심으로 구성한다.
- 어려운 명리 용어는 내부 판단 근거로만 사용하고, 본문 전면에 반복 노출하지 않는다.

## 3. MVP 범위

| 기능 | MVP 포함 여부 | 설명 | 확인 필요 |
| --- | --- | --- | --- |
| 관계 유형 선택 | 포함 | 연애, 부부, 가족, 직장, 친구 중 선택 | 기존 `CompatibilityRelationshipSlug`는 `lover`, `family`, `friend`, `partner` 4개라 `부부`, `직장` 세분화 방식 확인 필요 |
| 나/상대 생년월일시 입력 | 포함 | 기존 `UnifiedBirthInfoFields` 흐름 재사용 | 없음 |
| 저장 프로필 불러오기 | 포함 | 로그인 사용자는 내 프로필/가족 프로필 선택 | 기존 `/api/profile`, `/api/family-profiles` 재사용 |
| 비로그인 수동 입력 | 포함 | sessionStorage 기반 수동 궁합 흐름 재사용 | 결제 전환 시 입력값 보존 확인 필요 |
| 나/상대 16유형 성향 선택 | 포함 | 사용자가 알고 있는 유형을 직접 선택 | UI에서 공식 검사처럼 보이지 않게 표기 필요 |
| 8문항 성향 체크 | 포함 | 유형을 모를 때 간단 체크로 성향 축 추정 | 문항/점수 모델 신규 정의 필요 |
| 5축 점수 산출 | 포함 | 관계 온도, 표현 방식, 갈등 패턴, 생활 리듬, 회복 가능성 | 축 이름 최종 확정 필요 |
| 무료 결과 | 포함 | 관계 한 줄, 잘 맞는 지점, 어긋나는 지점, 오늘 행동 1개 | 없음 |
| 990원 깊이보기 | 포함 | 상세 관계 풀이와 실천 문장 제공 | 기존 `love-question` 상품 재사용 여부 확인 필요 |
| 결과 저장 | 포함 | 로그인 사용자는 보관함 저장 | 저장 대상이 `readings`인지 `paid_reading_snapshots`인지 확인 필요 |
| 공유 카드 | 포함 | 결과 한 줄/점수/관계 유형 중심 이미지 또는 카드 | 이미지 생성 없이 HTML 공유 카드부터 시작 권장 |
| AI 상담 연결 버튼 | 포함 | 결과 context를 대화방으로 전달 | `/dialogue`에 궁합 context 전달 방식 확인 필요 |

## 4. 제외 범위

MVP에서는 아래 항목을 제외한다.

- 공식 MBTI 검사처럼 보이는 UI
- 30문항 이상 정밀 검사
- 결혼/이별/재회 단정
- 부모/자녀 심층 해석
- 법률, 의료, 투자 판단
- 상대의 마음을 확정적으로 읽는 표현
- 벡터 DB 기반 RAG
- PDF 생성
- 가족 구성원 전체 관계 지도

주의: "공식 MBTI 검사", "MBTI 진단", "MBTI 심리검사" 같은 표현은 사용자 화면에서 사용하지 않는다. 문서상으로는 금지/제외 범위를 설명하기 위한 메타 표현으로만 둔다.

## 5. 사용자 플로우

### 5.1 진입

1. 홈, 궁합 메뉴, 또는 사주 결과 CTA에서 `달빛 성향궁합`으로 진입한다.
2. 관계 유형을 선택한다.
3. 내 정보를 입력하거나 저장된 내 프로필을 불러온다.
4. 상대 정보를 입력하거나 저장된 가족/상대 프로필을 불러온다.

### 5.2 성향 입력

1. 사용자가 나의 16유형 성향을 알고 있으면 직접 선택한다.
2. 사용자가 상대의 16유형 성향을 알고 있으면 직접 선택한다.
3. 모르면 `8문항으로 빠르게 확인`을 선택한다.
4. 8문항 체크는 정확한 진단이 아니라 성향 축을 추정하는 보조 입력으로 안내한다.

### 5.3 무료 결과

1. 결과 페이지로 이동한다.
2. 관계 한 줄 요약을 먼저 보여준다.
3. 5축 점수를 짧게 보여준다.
4. 잘 맞는 지점 2개, 자주 어긋나는 지점 2개를 보여준다.
5. 오늘 바꿔볼 행동 1개를 보여준다.
6. 990원 깊이보기, 공유, 저장, AI 상담 연결 버튼을 제공한다.

### 5.4 유료 깊이보기

1. 사용자가 `990원 깊이보기`를 누른다.
2. 결제 전 `/api/payments/prepare`로 중복 구매 여부를 확인한다.
3. 미구매면 Toss 결제로 이동한다.
4. 구매 완료 후 상세 결과 화면으로 이동한다.
5. 이미 구매했다면 결제창 대신 기존 상세 결과로 이동한다.

### 5.5 저장과 재열람

1. 로그인 사용자는 결과를 저장할 수 있다.
2. 유료 결과는 결제 당시 스냅샷으로 저장한다.
3. 보관함에서 열 때 현재 날짜 기준으로 재생성하지 않는다.

## 6. 입력 데이터

### 6.1 사주 입력 데이터

기존 `BirthInput` 흐름을 따른다.

- 이름 또는 표시명
- 양력/음력
- 생년월일
- 태어난 시간
- 시간 미상 여부
- 성별
- 출생지
- 표준시/진태양시 등 시간 규칙

### 6.2 성향 입력 데이터

신규로 필요한 데이터:

- 내 16유형 성향 코드
- 상대 16유형 성향 코드
- 내 성향 입력 방식: 직접 선택 또는 8문항 체크
- 상대 성향 입력 방식: 직접 선택 또는 8문항 체크
- 8문항 답변 원본
- 추정된 성향 축

확인 필요:

- 16유형 코드를 DB에 저장할지, 결과 스냅샷에만 저장할지 결정이 필요하다.
- 비로그인 사용자의 성향 입력값은 sessionStorage에 저장하는 것이 기존 수동 궁합 흐름과 맞다.

## 7. 결과 리포트 구조

### 7.1 무료 결과

1. 관계 한 줄
   - 예: "서로 끌리지만, 말의 속도를 맞출 때 편해지는 관계예요."
2. 관계 온도 점수
   - 0~100점
3. 5축 점수
   - 감정 온도
   - 말의 속도
   - 갈등 회복
   - 생활 리듬
   - 현실 조율
4. 잘 맞는 지점
   - 최대 2개
5. 자주 어긋나는 지점
   - 최대 2개
6. 오늘 해볼 행동
   - 1개
7. CTA
   - `990원 깊이보기`
   - `결과 저장`
   - `공유 카드 만들기`
   - `AI 상담으로 이어가기`

### 7.2 990원 깊이보기

1. 두 사람의 기본 결
2. 감정 표현 방식
3. 말투와 연락 리듬
4. 갈등이 커지는 패턴
5. 돈/생활/현실 조율 방식
6. 회복 대화 문장
7. 오늘부터 바꿔볼 행동 3개
8. 관계 유형별 조언
9. 참고 근거 접힘 영역

참고 근거 접힘 영역에는 전문 용어를 최소화한다. 필요한 경우에도 "사주 기준으로 본 참고 단서" 정도로 표현한다.

## 8. 무료/유료 구분

| 항목 | 무료 | 990원 깊이보기 |
| --- | --- | --- |
| 관계 한 줄 | 제공 | 제공 |
| 전체 점수 | 제공 | 제공 |
| 5축 점수 | 요약 제공 | 상세 제공 |
| 잘 맞는 지점 | 2개 | 구체 예시 포함 |
| 자주 어긋나는 지점 | 2개 | 갈등 패턴과 회복법 포함 |
| 오늘 행동 | 1개 | 3개 |
| 대화 예문 | 미제공 | 제공 |
| 저장 | 로그인 시 가능 | 결제 스냅샷 저장 |
| 공유 카드 | 기본 카드 | 상세 카드 또는 프리미엄 문구 포함 |
| AI 상담 연결 | 버튼 제공 | 결과 context 포함해 연결 |

확인 필요:

- 기존 `love-question` 990원 상품을 재사용할지, `personality-compatibility` 신규 상품을 추가할지 결정이 필요하다.
- 신규 상품을 추가하면 `product_entitlements` check constraint 마이그레이션이 필요하다.
- 1코인 해금과 990원 결제가 같은 상세 결과를 여는지 정책 결정이 필요하다.

## 9. 결제/권한 정책

MVP 권장안:

1. 무료 결과는 결제 없이 표시한다.
2. 990원 깊이보기는 `product_entitlements`를 기준으로 중복 결제를 막는다.
3. scope key는 두 사람의 입력값과 관계 유형을 기준으로 만든다.
4. 유료 결과는 `paid_reading_snapshots`에 결제 당시 내용을 저장한다.

권장 scope key 예시:

```text
compatibility:{relationship}:{selfBirthHash}:{partnerBirthHash}:{selfType}:{partnerType}
```

확인 필요:

- 해시 생성 위치와 개인정보 마스킹 정책을 정해야 한다.
- 상대 이름을 scope key에 직접 넣지 않는다.
- 비로그인 결과를 로그인 결제 후 어떤 user_id에 귀속할지 확인해야 한다.

## 11. QA 체크리스트

### 입력

- [ ] 로그인 사용자가 내 프로필을 바로 불러올 수 있다.
- [ ] 로그인 사용자가 가족/상대 프로필을 바로 불러올 수 있다.
- [ ] 비로그인 사용자가 내 정보와 상대 정보를 모두 입력할 수 있다.
- [ ] 생년월일 입력이 모바일에서 화면 밖으로 밀리지 않는다.
- [ ] 태어난 시간 미상 선택이 가능하다.
- [ ] 출생지 입력값이 결과까지 유지된다.
- [ ] 16유형 성향을 직접 선택할 수 있다.
- [ ] 8문항 체크로 성향을 추정할 수 있다.

### 결과

- [ ] 관계 한 줄 요약이 첫 화면에 나온다.
- [ ] 5축 점수가 모두 표시된다.
- [ ] 입력값을 바꾸면 점수와 문구가 달라진다.
- [ ] 명리 전문용어가 본문에 반복 노출되지 않는다.
- [ ] "무조건", "반드시", "절대", "파멸", "최악" 같은 표현이 없다.
- [ ] 결혼/이별/재회 확정 표현이 없다.
- [ ] 무료 결과와 유료 결과의 차이가 명확하다.

### 결제/권한

- [ ] 990원 깊이보기 클릭 전 중복 구매 여부를 확인한다.
- [ ] 이미 구매한 결과는 결제창 대신 상세 결과로 이동한다.
- [ ] 같은 두 사람/관계/성향 조합은 중복 결제되지 않는다.
- [ ] 다른 상대 또는 다른 관계 유형은 별도 결과로 구분된다.
- [ ] 결제 완료 후 스냅샷이 저장된다.
- [ ] 보관함 재열람 시 당시 결과가 다시 열린다.

### 공유/상담

- [ ] 공유 카드에는 개인정보가 과하게 노출되지 않는다.
- [ ] 공유 카드에는 관계 한 줄과 점수 정도만 표시한다.
- [ ] AI 상담 버튼은 결과 context를 전달한다.
- [ ] AI 상담 답변은 궁합 결과를 짧게 참고하되, 사주 원국 설명을 길게 반복하지 않는다.

### 모바일

- [ ] iPhone SE 폭에서 가로 스크롤이 없다.
- [ ] Galaxy S10급 기기에서 버벅임이 과하지 않다.
- [ ] CTA는 한 화면에 주 행동 1개, 보조 행동 1개 이하로 보인다.
- [ ] 하단 dock이 CTA를 가리지 않는다.

## 12. 구현 전 확인 필요 항목

| 항목 | 이유 |

| 990원 상품을 기존 `love-question`으로 재사용할지 신규 product로 만들지 | 신규 product는 DB check constraint 마이그레이션 필요 |
| 1코인 해금 지원 여부 | 원화 결제와 같은 entitlement를 공유해야 중복 결제를 막을 수 있음 |
| 결과 저장 위치 | 무료 결과는 session/readings, 유료 결과는 snapshot이 적절해 보이나 최종 결정 필요 |
| 16유형 성향 저장 정책 | 민감도는 낮지만 성향 데이터라 개인정보 정책에 포함 필요 |
| AI 상담 context 전달 방식 | `/dialogue` 기존 route/query/session 구조 확인 필요 |
| 공유 카드 구현 방식 | 이미지 생성 없이 HTML 카드부터 시작할지 결정 필요 |

## 13. MVP 완료 기준

- 로그인/비로그인 모두 성향궁합 무료 결과를 볼 수 있다.
- 저장 프로필을 클릭해 내 정보와 상대 정보를 채울 수 있다.
- 16유형 성향 직접 선택과 8문항 체크가 모두 가능하다.
- 결과가 두 사람의 사주 입력값과 성향 입력값에 따라 달라진다.
- 990원 깊이보기는 중복 결제 없이 열린다.
- 보관함에서 유료 결과를 당시 스냅샷으로 다시 볼 수 있다.
- 공유 카드와 AI 상담 연결 버튼이 동작한다.
- `npm run lint`, `npm run typecheck`, `npm test`, `npm run build` 기준을 통과한다.
