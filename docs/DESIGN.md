# 달빛인생 디자인 기본 정의

이 문서는 앞으로 달빛인생의 모든 UI/UX 변경에서 최우선으로 참고하는 디자인 기준서입니다.

## 1. 제품 방향

달빛인생은 “사주를 설명하는 사이트”가 아니라 “오늘 바로 눌러보는 운세 서비스”입니다.

사용자는 명리학을 공부하러 오는 것이 아니라 오늘의 마음, 연애, 돈, 일, 관계, 가족, 선택의 타이밍이 궁금해서 들어옵니다. 따라서 첫 화면과 결과 화면은 전문 설명보다 바로 누를 수 있는 메뉴와 짧게 이해되는 풀이를 먼저 보여줘야 합니다.

## 2. 브랜드 원칙

- 사이트명은 `달빛인생`으로 통일합니다.
- `달빛선생`, `명리 기준서`, `엔진 기술서`, `판정 근거` 같은 이전 리포트형 표현은 전면 메뉴에서 낮춥니다.
- 홈과 주요 진입면은 무료 오늘운세, 무료 타로, 550원/990원 소액 풀이 중심으로 설계합니다.
- 사용자가 결제 압박을 먼저 느끼지 않도록, 결제는 무료 풀이 후 “더 보고 싶은 질문”으로 자연스럽게 이어집니다.
- 사주 전문용어는 본문에 반복 노출하지 않고, 필요한 경우 `풀이 기준 보기` 같은 접힘 영역으로 분리합니다.

## 3. 시각 톤

전체 톤은 흰 배경, 블랙 텍스트, 핑크 포인트입니다.

달빛인생은 고급 리포트 사이트보다 모바일 운세 앱에 가깝게 보여야 합니다. 화면은 가볍고 빠르게 눌러볼 수 있어야 하며, 카드와 버튼은 명확하고 눈에 잘 들어와야 합니다.

### 핵심 색상

| 역할 | 색상 | 사용처 |
| --- | --- | --- |
| 배경 | `#ffffff` | 기본 페이지 배경 |
| 잉크 | `#111114` | 제목, 주요 텍스트 |
| 보조 텍스트 | `rgba(17, 17, 20, 0.64)` | 설명, 보조 문장 |
| 메인 핑크 | `#ff4f9a` | CTA, 선택 상태, 핵심 강조 |
| 진한 핑크 | `#d81b72` | 활성 탭, 가격, 강한 강조 |
| 연핑크 | `#fff0f7` | 보조 배경, 선택 카드 |
| 라인 | `rgba(17, 17, 20, 0.08)` | 카드 구분선 |
| 로그인 옐로 | `#ffd54a` | 상단 로그인 버튼 |

### 피해야 할 톤

- 달빛선생 시절의 다크 네이비, 골드, 리포트형 무거운 패널
- 엔진 문서처럼 보이는 회색 표와 기술 메타데이터
- 설명이 많은 랜딩페이지형 구성
- 결제 가격이 첫 화면에서 과하게 압박되는 구성

## 4. 타이포그래피

- 기본 글꼴은 `Noto Sans KR`, `Apple SD Gothic Neo`, `Malgun Gothic`, system-ui, sans-serif 계열을 사용합니다.
- 장식성 제목, 이벤트 배너, 한자 표기도 별도 명조 계열로 분리하지 않고 같은 고딕 계열 안에서 굵기와 색으로 위계를 만듭니다.
- 한글 제목은 줄바꿈이 어색하지 않도록 `word-break: keep-all`, `line-break: strict`를 우선합니다.
- 글자 간격은 기본적으로 `0`을 유지합니다.
- 모바일 본문은 15px 이상, 주요 버튼은 14px 이상을 권장합니다.
- 시니어 사용자를 고려해 대비를 높이고, 너무 작은 설명문을 핵심 정보에 쓰지 않습니다.

## 5. 홈 화면 구조

홈은 설명 페이지가 아니라 앱형 진입 화면입니다.

권장 순서:

1. 상단 앱 헤더: 작은 브랜드 로고, 로그인 버튼, 메뉴 버튼
2. 메인 배너: 오늘 바로 눌러볼 운세 또는 이벤트성 운세
3. 카테고리 칩: 전체, 사주·명리, 운세·택일, 상담
4. 2열 서비스 카드: 사주, 궁합, 대운, 택일, 오늘운세, 타로, 띠운세, 상담
5. 하단 앱 도크: 홈, 사주추가, 무료운세, 대화방, 보관함

홈에서 크게 노출하지 않을 것:

- 긴 서비스 소개
- 신뢰장치 대형 섹션
- 판정근거, 엔진, 기준서 설명
- 결제 상품을 먼저 밀어붙이는 가격표

## 6. 네비게이션

### 모바일 하단 도크

기본 탭은 다음 5개입니다.

- 홈
- 사주추가
- 무료운세
- 대화방
- 보관함

`무료운세`는 가운데 주요 액션으로 더 강하게 보여줍니다. 하단 도크는 항상 앱처럼 직관적으로 보여야 하며, 탭 이름은 짧고 명확해야 합니다.

### 상단 헤더

- 배경은 흰색입니다.
- 왼쪽에는 작은 브랜드 잠금 로고와 `달빛인생`을 둡니다.
- 오른쪽에는 로그인 또는 보관 버튼을 둡니다.
- 기술적 보기 방식 전환, 개발자용 토글, 복잡한 보조 메뉴는 상단에 두지 않습니다.

## 7. 카드와 버튼

### 카드

- 홈 카드는 쇼핑몰 상품 카드처럼 즉시 이해되어야 합니다.
- 모바일에서는 2열 카드가 기본입니다.
- 각 카드에는 큰 시각 요소, 가격 또는 무료 표시, 상품명, 짧은 설명, CTA가 들어갑니다.
- 카드 안 설명은 1줄에서 2줄 안에 끝냅니다.

### 버튼

- 주 행동 버튼은 핑크 또는 블랙으로 명확하게 보입니다.
- 한 섹션에는 주 행동 1개, 보조 행동 1개까지만 둡니다.
- 버튼 문구는 `해설 보러가기`, `무료운세 보기`, `사주 입력하기`처럼 사용자의 다음 행동을 직접 말합니다.
- `자세히 알아보기`, `이용안내 보기` 같은 설명형 버튼은 홈 첫 화면에서 줄입니다.

## 8. 결과 화면 구조

결과 화면은 긴 리포트보다 “먼저 이해되는 운세 카드”가 우선입니다.

기본 순서:

1. 지금 핵심 한 줄
2. 오늘 또는 이번 달 조언
3. 재물, 연애, 일, 건강 카드
4. 더 보고 싶은 분야 선택
5. 550원/990원 소액 풀이 또는 저장/대화 유도
6. 필요한 경우 접힘형 풀이 기준

전문용어는 결과 본문에서 반복하지 않습니다. 예를 들어 격국, 용신, 합충 같은 기준 정보는 일반 사용자가 읽는 흐름을 방해하지 않도록 작게 분리합니다.

## 9. 결제와 상품 노출

소액 상품은 문제 중심으로 보여줍니다.

예시:

- 오늘 자세히 보기 550원
- 연애 마음 확인 990원
- 돈이 새는 패턴 보기 990원
- 일/직장 흐름 보기 990원
- 중요한 날 확인 990원

가격은 무료 풀이 뒤에 자연스럽게 노출합니다. 홈 첫 화면에서 결제 압박을 주는 큰 가격표는 피합니다.

이미 구매한 항목은 다시 결제 버튼을 보여주지 않고, 바로 열람 상태로 보여줘야 합니다.

## 10. 캐릭터와 선생님 시스템

십이간지 캐릭터와 선생님은 메뉴를 쉽게 기억하게 만드는 장치입니다.

원칙:

- 기능별 선생을 메뉴와 상품에 연결합니다.
- 아직 만들지 않은 기능은 준비 중으로 표시하되, 메뉴 구조는 확장 가능하게 둡니다.
- 캐릭터는 귀엽고 직관적으로 쓰되, 풀이 자체의 신뢰를 해치지 않게 과하게 장식하지 않습니다.

## 11. 접근성과 모바일 기준

- 모바일 360px, 390px, 768px에서 먼저 확인합니다.
- 하단 도크에 내용이 가리지 않도록 충분한 safe-area 여백을 둡니다.
- 제목과 버튼 텍스트는 줄바꿈이 깨지지 않아야 합니다.
- 한 화면에서 사용자가 첫 행동을 3초 안에 찾을 수 있어야 합니다.
- 시니어 사용자를 위해 대비, 버튼 크기, 문장 길이를 항상 점검합니다.

## 12. 구현 원칙

- 전역 스타일은 `src/app/globals.css`의 달빛인생 토큰을 우선합니다.
- 새 컴포넌트는 기존 공통 레이아웃과 네비게이션 구조를 재사용합니다.
- 페이지별 수제 다크 패널이나 골드 패널을 새로 만들지 않습니다.
- 디자인 변경 전에는 이 문서와 현재 홈/하단 도크 기준을 먼저 확인합니다.
- 디자인 판단이 애매하면 “오늘 바로 눌러보는 운세 서비스”에 더 가까운 선택을 합니다.

## 13. 금지 표현과 후순위 표현

전면 UI에서 피할 표현:

- 엔진 기술서
- 저장 메타데이터
- evidence_json
- 판정 근거
- 명리 기준서
- 무제한 혜택
- 기술 검증

필요할 때 바꿔 쓰는 표현:

- `풀이 기준 보기`
- `왜 이렇게 봤나요?`
- `조금 더 깊게 보기`
- `다시 보기`
- `보관하기`
- `오늘 바로 보기`

## 14. QA 체크 기준

디자인 변경 후 반드시 확인합니다.

- 첫 화면에서 무료 오늘운세와 타로 진입이 바로 보이는가
- 버튼이 누르고 싶게 보이는가
- 설명이 너무 많지 않은가
- 다크 네이비/골드/리포트형 잔재가 남지 않았는가
- 모바일 하단 도크가 콘텐츠를 가리지 않는가
- 상품 노출이 결제 압박처럼 보이지 않는가
- 달빛선생 시절 문구가 남아 있지 않은가


## 15. Ganji Design System v1

현재 production 기준 공통 컴포넌트는 `src/components/gangi/gangi-ui.tsx`를 우선합니다.

핵심 API:

- `GangiPageHeader`: 모바일 앱형 서브페이지 상단 헤더
- `GangiIntro`: 첫 설명 영역. 한 화면에서 바로 이해되는 짧은 제목과 설명만 둡니다.
- `GangiSection`: 흰 카드 또는 연핑크 카드형 섹션. 새 섹션의 기본 표면입니다.
- `GangiActionRow`: 한 섹션 안의 CTA 묶음. 주 행동 1개, 보조 행동 1개를 기본으로 합니다.
- `GangiListLink`: 상품, 메뉴, 다음 행동을 한 줄 카드로 보여주는 기본 리스트 컴포넌트입니다.
- `GangiMiniCard`: 결과 요약, 단계, 짧은 근거를 2~3개로 나눠 보여주는 작은 카드입니다.

새 화면을 만들 때는 shadcn 컴포넌트를 직접 조합하기보다 위 컴포넌트로 먼저 구성합니다. 기존 `SectionSurface`, `SectionHeader`, `FeatureCard`, `ProductGrid`는 아직 남은 화면을 위한 호환 계층으로 보고, 신규 핵심 플로우에는 되도록 추가하지 않습니다.

## 16. Legacy Style Cleanup Policy

전역 CSS는 한 번에 삭제하지 않습니다. 아래 순서로 줄입니다.

1. 핵심 플로우에서 `moon-action-*`, `moon-lunar-panel`, `app-starfield` 직접 사용을 먼저 제거합니다.
2. 사용처가 0개가 된 클래스만 `globals.css`에서 삭제합니다.
3. `--app-gold` 계열 변수는 당장 제거하지 않고 핑크 alias로 유지합니다. 남은 화면이 모두 `--app-pink`로 이동한 뒤 삭제합니다.
4. 전문/고전 페이지에 남은 레거시 문법은 핵심 전환 흐름 뒤에 정리합니다.
5. 삭제 전에는 반드시 `npm run typecheck`, `npm test`, `npm run build`를 통과시킵니다.
