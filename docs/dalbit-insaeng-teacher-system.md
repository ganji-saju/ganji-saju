# 달빛인생 선생님 시스템 설계

## 목적

달빛인생은 어려운 명리 설명보다 사용자가 바로 누를 수 있는 운세 서비스를 먼저 보여준다. 선생님 시스템은 메뉴와 상품을 기능명 중심이 아니라 “누가 어떤 고민을 봐주는가” 중심으로 정리하기 위한 브랜드 구조다.

## 기본 원칙

- 첫 화면에서는 무료 오늘운세와 무료 타로를 먼저 보여준다.
- 선생님 캐릭터는 메뉴 탐색과 상품 선택을 쉽게 만드는 라벨로 사용한다.
- 실제 기능이 없는 선생님은 준비 중 상태로만 표시하고 결제 흐름에 연결하지 않는다.
- 준비 중 선생님은 모두 `/guide?teacher=...` 안내 링크로만 연결한다.
- 전문용어, 기준서, 엔진, 판정근거 표현은 안내 영역이나 상세 영역으로 낮춘다.
- 소액 상품은 선생님 캐릭터와 사용자의 질문을 함께 보여준다.
- 기능을 열 때는 `src/content/moonlight.ts`의 `status`와 `href`를 먼저 바꾼다.

## 구현 위치

- 선생님 데이터: `src/content/moonlight.ts`
- 홈 메뉴 반영: `src/app/page.tsx`
- 가격 페이지 상품/선생 연결: `src/app/pricing/page.tsx`
- 멤버십 소액 상품 반영: `src/app/membership/page.tsx`
- 상단/모바일 메뉴 설명: `src/features/shared-navigation/site-header.tsx`
