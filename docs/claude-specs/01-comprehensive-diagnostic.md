## 0. 한눈에 보는 결론

ganjisaju.kr은 **명리 정보 설계가 잘 되어 있는 사이트**입니다. 6분야 게이지, 시기 라벨 컬러 코딩, 월간 캘린더 시각화, 안심하고 보기 섹션, 분야운 카드 구조 — 이런 자산들은 운세톡톡·사주아이 같은 경쟁사와 비교해도 떨어지지 않아요.

문제는 **콘텐츠 생성 엔진(L2 계산 → L4 표시 사이)이 망가져 있다**는 점입니다. 구체적으로:

1. **템플릿 변수 누수 버그 6개** — 본문에 그대로 노출 (5분~1시간 안에 고칠 수 있음)
2. **"결단과/안정과/열정과" 네이밍이 한국어 문법을 깨뜨림** — 가장 큰 가독성 문제
3. **콘텐츠 중복** — 같은 문장이 9장 리포트 + 세운 카드 + 상세 탭에 4~7번씩 등장
4. **한자 본문 노출** — 거의 모든 페이지
5. **영어 enum 누수** — `weak`, `balanced`, `timing`이 한국어 본문에 박힘
6. **한자→한글 매핑 오류** — `己`가 어떤 곳은 "근", 어떤 곳은 "기"로 들쭉날쭉

이 6가지를 순서대로 고치면 사이트의 체감 품질이 단계별로 올라갑니다. **버그 6개부터 고치는 게 ROI 최고**입니다 (반나절~1주, 사용자 신뢰도 즉시 회복).

## 2. 잘 하고 있는 것들 — 반드시 유지할 자산

진단 전에 짚어둘 강점들. 개선 작업 중에 실수로 망가뜨리지 않도록 주의해야 합니다.

### 2-1. UI 자산

- **6분야 점수 게이지** (총평 / 상세) — 운세톡톡 5종 대비 우위. 색상 분리도 좋음.
- **시기 라벨 컬러 코딩** — 성장기/전환기/표현기/기반기/결정기/준비기 6단계로 인생 분류. 직관적.
- **"지금" 태그 + 교운기 진입/마무리 표시** — 정보 위계 우수.
- **월간 지도 캘린더 시각화** (일진 탭) — 31일이 결정일/좋은날/주의날/보통날로 색상 분리되어 한눈에 들어옴.
- **12개월 흐름 레일** (세운 탭) — 1~12월을 카드 12개로 펼치고 좋음/확인 라벨 + 분야 태그.
- **분야운 카드 구조** (세운 탭 분야별) — 통하는 장면 / 엉키는 장면 / 표현 방법 / 거리 조절 같은 **카드 슬롯 설계가 매우 좋음**.
- **PEAK / PITFALL 월 표시** — 1월 PEAK, 2월 PITFALL 같은 강조 라벨.

### 2-2. 정보 설계 자산

- **9장 평생 리포트의 챕터 구조 자체** — 1장 성향 → 2장 균형 → 3장 보완 → 4장 관계 → 5장 재물 → 6장 직업 → 7장 건강 → 8장 대운 → 9장 평생 전략. 정보 흐름이 자연스러움.

## 3. 전 사이트 공통 문제 진단

### 3-1. 템플릿 변수 누수 버그 (P0 — 즉시 수정)

가장 즉각적으로 사용자 신뢰를 깎는 버그들입니다. 모두 명백한 코드/템플릿 결함이라 비교적 빠르게 수정 가능.

**버그 ①: "내 내 내 사주표"** *(2차 스크린샷 4·10·13, 3차 스크린샷 11)*

> "생활 전체의 템포가 어긋나기 쉬운 **내 내 내** 사주표입니다."
> "열정과 기운이 앞에 서는 **내 내** 사주표이라…"

`{{user_pronoun}}` 변수가 2~3번 중복 치환됨. `"내 사주를"` 한 번만 들어가야 하는 자리. **이건 사이트 첫인상에서 가장 먼저 눈에 들어오는 버그.** 5분 안에 수정 가능.

**버그 ②: "근 내 핵심 기질" + "근표" 태그** *(2차 스크린샷 12, 3차 스크린샷 5)*

> "**근 내** 핵심 기질의 추진력과…" *(2장 한 줄 요약)*
> 태그: **근표** *(1월 카드 하단)*

한자 `己`의 한국어 음은 **"기"**입니다 ("기토" 일간). 다른 곳에는 "**己** 내 핵심 기질"로 한자 그대로 나오고, 어떤 곳에는 한글 변환하다가 "근"으로 잘못 변환됨. 한자→한글 매핑 테이블에 `己 → 근`이라는 오타가 있는 것으로 추정. 모든 천간/지지 매핑을 한 번 점검해야 합니다.

**버그 ③: 영어 enum 누수** *(2차 스크린샷 9, 3차 스크린샷 11)*

> "결단과 3.1% · **weak** · 0.4점"
> "시작과 13.3% · **balanced** · 1.7점"
> "자리를 바꾸거나 생활 패턴을 크게 손댈 때는 명분과 **timing**을 같이 보는 편이 좋습니다."

내부 enum 값(`weak`, `balanced`, `timing`)이 한국어 표시 매핑 없이 그대로 출력됨. 한글 매핑 테이블 추가 필요:
- `weak → 약함`
- `balanced → 균형`
- `strong → 강함`
- `timing → 시점` 또는 `타이밍`

**버그 ④: 세운 본문의 종결문 접속 오류** *(3차 스크린샷 1, 가장 심각)*

> "2026년은 丙午의 흐름 아래에서 **기준 보완이 필요해, 일은 속도보다 완성도를 먼저 챙기세요과 기준 보완이 필요해, 관계는 거리와 표현 순서를 맞추는 게 핵심입니다를 같이 조율해야** 한 해가 안정적으로 풀립니다."

템플릿이 `"{판단A}과 {판단B}를 같이 조율해야"` 패턴인데, 판단 A·B가 "~챙기세요", "~핵심입니다" 같은 **완전한 종결문**으로 들어가버려서 "챙기세요과", "핵심입니다를" 같은 비문 발생. 그리고 "기준 보완이 필요해"가 두 판단에 중복.

수정 방향: 판단 A·B는 **명사구**로 끝나야 함. 예: "일은 완성도 우선" + "관계는 거리·표현 순서 정리" → "**일의 완성도와 관계의 거리·표현 순서를 같이 조율해야 안정적으로 풀립니다.**"

**버그 ⑤: "커안쪽 결만"** *(3차 스크린샷 4, 5월 카드)*

> "5월: 바꾸고 싶은 마음이 **커안쪽 결만** 시험 운전이 먼저인 달입니다."

원래 의도된 표현은 "**커지지만**" 또는 "**커지는 때이지만**"으로 추정. `{verb}{쪽}{만}` 같은 변수 자리가 잘못 결합. 5월 카드 본문 템플릿을 다시 봐야 합니다.

**버그 ⑥: "기사일주, 따뜻하게 익히는 흙·정인격의 결을 가진 사주입니다"** *(1차 스크린샷 총평)*

> "구직 중이신 간지사주 테스트님, 기사일주, 따뜻하게 익히는 **흙·정인격**의 결을 가진 사주입니다."

"흙"은 일상어, "정인격"은 명리 전문 용어. 한 문장에 일상어 풀이와 한자 용어가 충돌. 또 "흙·정인격" 같이 **점(·)으로 일상어와 명리어를 묶는 표기**가 본문 곳곳에 등장. 가독성 깨짐. 일상어 라벨로 통일해야 (예: "정인격" → "돌봄·후원의 결").

### 3-2. "X과" 네이밍 (오행) — 가독성의 가장 큰 적

사이트 전체 톤의 가장 큰 적입니다. 의도는 좋지만 한국어 문법을 깨뜨리고 있습니다.

**현재 오행 라벨 (추정)**:

| 오행 | 사이트 라벨 | 등장 빈도 |
|------|------------|----------|
| 木 (목) | **시작과** | 자주 |
| 火 (화) | **열정과** | 매우 자주 |
| 土 (토) | **안정과** | 매우 자주 |
| 金 (금) | **결단과** | 매우 자주 |
| 水 (수) | (지혜과?) | 미확인 |

**왜 문제인가**:

한국어에서 "**-과**"는 접속 조사("X와/과")입니다. 그래서 본문에 "**결단과 기운**"이 나오면 한국인은 자동으로 "**결단(과) 기운**" = "**결단과 기운 두 가지**"로 파싱합니다. 의도와 다른 의미가 만들어져요.

**실제 사이트 예시**:

> "**결단과** 축이 비거나 **열정과** 기운이 과속할 때, 장점이 곧 피로와 고집으로 바뀌기 쉽습니다."
> ↑ 한국인의 자동 파싱: "결단(과) 축이 비거나, 열정(과) 기운이 과속할 때"

> "**결단과** 안정과 **보완 축**이 살아나는 선택은 길게 보면…"
> ↑ "결단과 안정과 보완 축" — 무엇과 무엇이 연결된 건지 모호

이 라벨이 본문에 100번 이상 등장하기 때문에, 사이트 전체가 "약간 어색한 한국어"라는 인상을 줍니다.

**수정안 (3가지 옵션, 추천 순)**:

**(A) 자연 비유로 교체** ← **가장 추천**. 사주의 본질(오행 = 자연의 다섯 모습)과 일치, 시그니처 톤 강화.

| 오행 | 자연 비유 라벨 | 한 단어 키워드 |
|------|--------------|-------------|
| 木 | **새싹의 결** | 자라남 |
| 火 | **햇살의 결** | 밝게 퍼짐 |
| 土 | **흙의 결** | 담아내고 기름 |
| 金 | **쇠의 결** 또는 **칼날의 결** | 단단함·자름 |
| 水 | **물의 결** | 흐름·깊이 |

본문 예시 (BEFORE/AFTER):

> **BEFORE**: "결단과 축이 비거나 열정과 기운이 과속할 때, 장점이 곧 피로와 고집으로 바뀌기 쉽습니다."
>
> **AFTER**: "쇠의 결이 약하거나 햇살의 결이 너무 강할 때, 장점이 곧 피로와 고집으로 바뀌기 쉽습니다."

**(B) "X-력" 또는 "X의 결" 접미사로 교체** ← 점진적 마이그레이션 시 추천

- 결단과 → **결단력** 또는 **결단의 결**
- 안정과 → **안정감** 또는 **안정의 결**
- 열정과 → **열정** 또는 **활력**
- 시작과 → **시작의 힘** 또는 **새 출발의 결**

**(C) 한글 오행 그대로** ← 가장 단순, 명리 정확성 유지

- 木 → **목** (첫 등장 시 "새싹의 결" 같이 짧은 풀이)
- 火 → **화** (첫 등장 시 "햇살의 결")
- 등등

**작업 분량**: 사이트 전체 본문·카드·태그에서 "결단과/안정과/열정과/시작과" 일괄 치환. 한 번 정하면 30~50개 위치 수정. 1~2일 작업.

### 3-3. 한자 본문 노출

거의 모든 페이지에 한자가 본문에 그대로 나옵니다.

**자주 등장하는 한자**:
- 천간: `甲乙丙丁戊己庚辛壬癸` (특히 일간 `己`)
- 지지: `子丑寅卯辰巳午未申酉戌亥`
- 대운 표기: `丙申`, `丙午`, `丁酉`, `戊戌`, `己亥`, `庚子`, `辛丑`, `壬寅`, `癸卯`
- 오행: `木火土金水`
- 도식: `大運`, `四柱`, `年`, `月`
- 기타: `交運期`

**규칙 제안**:

| 위치 | 한자 허용 여부 |
|------|-------------|
| 사주팔자 8글자 카드 | ✅ 허용 (정체성) |
| 大運 같은 도식 헤더 | ⚠️ 한글 "대운"으로 |
| 본문 ("丙申 대운에는…") | ❌ 금지 → "지금 진행 중인 22-31세의 10년은…" |
| 실천 카드 "金 기운이 약해…" | ❌ 금지 → "쇠의 결이 약해…" |
| 마무리 "丙申 대운이 진행 중인 지금…" | ❌ 금지 |

**자동 검증**: 후처리 단계에서 정규식 `/[\u4e00-\u9fff]/g`로 본문에서 한자 검출 → 자동 치환 또는 재생성. `daewoon-llm-spec.md` §6에 코드 있음.

### 3-4. 영어 단어 누수

3-1의 ③번 버그와 같은 맥락이지만 별도 강조. 한국어 사주 사이트에 `weak`, `balanced`, `timing` 같은 영어 단어가 본문에 박히면 "기획 안 된 사이트" 인상을 줍니다.

**전수 점검 필요한 enum**:
- 강약: `weak / balanced / strong` → 약함 / 균형 / 강함
- 시점: `timing` → 타이밍 또는 시점
- 길흉: `good / caution / pitfall / peak` (이미 일부 한글)
- 단계: `early / mid / late`
- 분야: `career / love / money / health / relationship`

내부 enum과 사용자 표시 라벨을 **반드시 분리**해서, i18n 패턴으로 관리하는 게 정석.

### 3-5. 한자→한글 매핑 오류

위 3-1의 ②번 버그 (`己 → 근`). 다른 천간·지지에도 동일한 오류가 있을 가능성. 전수 점검 권장:

| 천간 | 정확한 한국어 음 | 의심 항목 |
|------|--------------|----------|
| 甲 | 갑 | |
| 乙 | 을 | |
| 丙 | 병 | |
| 丁 | 정 | |
| 戊 | 무 | |
| **己** | **기** | ← "근"으로 오기 확인됨 |
| 庚 | 경 | |
| 辛 | 신 | |
| 壬 | 임 | |
| 癸 | 계 | |

지지 12자도 동일하게 점검.

### 3-6. 콘텐츠 중복 — 같은 문장이 4~7곳에 등장

이게 사이트의 **가장 구조적인 문제**입니다. 단순 버그가 아니라 콘텐츠 생성 엔진의 한계에서 나오는 문제.

**최악의 중복 케이스 — "배움과 도움을 크게 쓰는 편이라 도움을 받고 다시 키워내는 힘이 오늘 선택에 먼저 드러납니다."**

이 문장은 사용자의 **정인격**(돌봄·후원의 별) 특성을 풀어 쓴 것인데, 다음 자리에 모두 등장합니다:

1. **상세 탭 1장** (타고난 성향) - 도입 문장
2. **상세 탭 4장** (관계 패턴) - 도입 문장
3. **상세 탭 5장** (재물 감각) - 도입 문장
4. **상세 탭 6장** (직업 방향) - 도입 문장
5. **상세 탭 9장** (평생 활용 전략) - 본문
6. **세운 탭 직장운 카드** "올해 핵심"
7. **세운 탭 재물운 카드** (추정)
8. **세운 탭 연애운 카드** "올해 핵심"

**8군데에 같은 한 문장이 들어 있는 셈**입니다. 사용자가 사이트를 위에서 아래로 읽으면 같은 문장을 8번 봅니다.

**또 다른 중복 케이스 — "표현은 충분히 살아 있고 기준은 비기 쉬워, 기준 · 정리를 챙길수록 하루가 덜 흔들립니다."**

- 상세 4장 도입
- 상세 5장 도입 (4장과 완전 동일)
- 상세 9장 평생 활용 전략
- 세운 탭 재물운 카드 올해 핵심
- 세운 탭 관계운 카드 올해 핵심

이런 식의 8~10개 문장이 사이트 전체에서 4~8번씩 반복됩니다.

**근본 원인**: 사주 데이터(정인격, 신강, 火 과다 등)에서 "도출되는 한 문장 풀이"가 명리 룰 테이블에 박혀 있고, 그걸 챕터·카드 슬롯마다 직접 갖다 붙이는 구조. **각 챕터/카드는 이 데이터를 자기만의 렌즈로 다시 풀어 써야** 하는데, 그 변환층이 없음.

**해결 방향**: 

이전 답변에서 짚었던 L2.5 + L3 + L3.5 아키텍처 그대로. 각 챕터·카드마다 LLM이 같은 사주 데이터를 **그 챕터/카드의 주제 렌즈로** 재해석하도록 프롬프트를 분리. `daewoon-llm-spec.md`의 8섹션 분리 호출 패턴을 9개 챕터 + 6개 분야 카드 + 12개 월 카드 + N개 일진 카드 모두에 적용해야 합니다.

작업 분량이 큰 P2~P3 작업이지만, **이게 없으면 어떤 톤 개선을 해도 "왜 같은 얘기를 또 하지?"라는 인상은 사라지지 않습니다.**

## 4. 페이지별 구체적 문제

### 4-1. 총평 탭

**잘된 것**:
- 한 줄 요약 카드의 "표현이 앞서고 정리가 비기 쉬운 날입니다" 같은 컨셉은 좋음
- 6분야 게이지 ✓
- 사주팔자 8글자 카드 정체성 유지 ✓
- 컨텍스트 칩(솔로/구직 중/사업·이직) ✓

**문제**:
- "기질 안정 · 강점 표현 45% · 보완 정리 3%" 태그 — **수치의 의미 불명**. 분모가 무엇인지 사용자가 모름. 툴팁 또는 한 줄 설명 추가 필요.
- "기사일주, 따뜻하게 익히는 흙·정인격의 결을 가진 사주" — 일상어 "흙"과 명리어 "정인격"을 점(·)으로 묶는 표기. 어색.
- 본문에 `己巳`, `丙申`, `丙午`, `癸巳`, `金`, `土` 한자 노출

### 4-2. 대운 탭 (상세 탭 8장과 별도 페이지)

**잘된 것**:
- 일주 카드 + 띠 표시
- 9개 대운 가로 스크롤 카드
- 시기 라벨(성장기/전환기 등) 컬러 코딩
- 8섹션 구조 (호명/제목/본문/멘탈/관계/돈커리어/실천4단/마무리)

**문제** (이전 답변에서 상세히 다룸):

- **9개 대운 본문이 거의 동일한 템플릿**: "X 대운에는 천간의 [열정/안정/시작/결단]과 · 지지의 [열정/안정/시작/결단]과 결이 함께 작동합니다…" 패턴이 9번 반복.
- **관계·돈커리어·실천 4단 중 일부 섹션은 9개 대운에 같은 문장 그대로**
- **챕터 제목 중복**: "대박 나는 비책, 이것만 지키면 흐름이 바뀝니다"가 을미·임인·계묘 3개 대운에 동일 등장 (fallback 누수)
- **자극적 톤**: "⚠️ 암흑기 주의보!", "텅장을 면해요" — 사이트의 "공포 표현 없음" 정책과 충돌
- 본문에 한자 9개 대운 모두 노출 (`甲午`, `乙未`, `丙申`…)

**해결**: 이미 작성된 `daewoon-llm-spec.md` 스펙 그대로 적용. 입력 JSON을 9개 대운 각각 다르게 만들고, 섹션별 호출.

### 4-3. 상세 탭 (9장 평생 리포트)

**잘된 것**:
- 9장 챕터 구조 자체는 정보 흐름이 자연스러움
- "한 장 요약 → 본문 → 카드 분리" 위계
- 카드 슬롯 라벨링 (반응 방식 / 강점 환경 / 무너지기 쉬운 패턴 / 거리감 / 감정 표현 방식 등)이 좋음
- "풀이 배경 보기" 접힘 메뉴로 깊이 선택 가능

**문제**:
- **챕터 도입 패턴이 일률적**: 9개 챕터 중 거의 다 "차분히 흐름을 읽어보면…"으로 시작
- **챕터 간 본문 중복**: 4장(관계)·5장(재물) 도입 문장 100% 동일. 1·4·5·6·9장에 "배움과 도움을 크게 쓰는 편이라…" 반복.
- **9장(평생 활용 전략)이 1~7장의 복사·붙여넣기**: 9장 본문은 거의 모두 다른 챕터에서 이미 본 문장. 진짜 "평생 활용 전략"이 되려면 사주 전체를 관통하는 *3~5개 평생 원칙*이 도출되어야 함.
- "내 내 내 사주표" 버그 (3-1 ①)
- "근 내 핵심 기질" 버그 (3-1 ②)
- 한자 노출, "결단과/안정과/열정과" 네이밍 문제

**우선 작업**:
1. 9장을 진짜 synthesis로 다시 쓰기 — 평생 의사결정 원칙 3~5개
2. 4·5장 도입 문장 분화
3. 9개 챕터 도입 패턴 다양화

### 4-4. 세운 탭 (올해 2026 흐름) — 이번 batch에서 처음 확인

**잘된 것**:
- 1장 올해 흐름 → 2장 어떤 선택 → 3장 12개월 흐름의 단계적 구조
- **연간 지도 시각화** (8개월 진행/4개월 확인/0개월 정리)
- **12개월 흐름 레일** — 각 월 카드에 좋음/확인 + 분야 태그 (생활 리듬·관계, 일·돈 등)
- **PEAK / PITFALL 강조** (1월 PEAK, 2월 PITFALL)
- **상하반기 먼저 볼 것** 카드
- **올해 핵심 키워드 5개** (기준 회복/현실 정돈/표현 절제/인연 조율/루틴 유지) — 매우 잘 만든 chip

**문제**:
- **세운 본문의 종결문 접속 버그** (3-1 ④): "챙기세요과 ~ 핵심입니다를 같이 조율해야" 비문
- **5월 카드 "커안쪽 결만" 버그** (3-1 ⑤)
- **분야운 카드의 "올해 핵심" 문장이 9장 리포트의 도입 문장과 동일** — 같은 문장이 또 8군데에서 반복
- "丙午", "丙申", "timing" 등 한자·영어 노출
- 월별 카드 텍스트 패턴이 유사: "준비한 결정을 밖으로 꺼내기 좋습니다"가 여러 월에 반복

**기회**: 분야운 카드의 구조(통하는 장면/엉키는 장면/표현 방법/거리 조절)는 **사이트에서 가장 잘 설계된 카드 슬롯**입니다. 이 그릇은 그대로 두고 채우는 텍스트만 다시 생성하면 됩니다.

### 4-5. 일진 (날짜별 행동 힌트)

**잘된 것**:
- **월간 지도 색상 캘린더** — 결정일/좋은 날/주의 날/보통 날 색 분리
- **날짜별 카드** — 일진 + 신살 태그(공망살/망신살/역마살) + 점수 + 오늘 할 일
- **사용자 이름 자연 호명**: "간지사주 테스트 님의 사주에 강한 결집의 기운이 작용합니다"
- **안심하고 보기 섹션** — 사이트 정체성의 핵심

**문제**:
- 일부 일진 카드 본문이 짧고 일반론적 ("같은 계절의 기운이 합쳐져 일관된 흐름이 형성되는 날입니다. 명확한 목표 하나에 올인하세요.")
- 신살 라벨이 그대로 노출 (공망살·망신살·역마살) — 일상어 풀이 함께 보여줄 것

**기회**: 일진은 LLM보다 결정론적 매칭(이전 대화에서 짠 250문장 라이브러리)이 더 빠르고 정확. 일진 라이브러리를 사전에서 받아와 채우는 방식 권장.

### 4-6. 분야운 카드 (직장/재물/연애/관계)

**잘된 것** (사이트에서 가장 잘 설계된 부분):
- 점수 + 분야명 (직장 72점, 재물 75점, 연애 65점, 관계 64점)
- 카드 슬롯: **올해 핵심 / 통하는 장면 또는 기회 장면 / 엉키는 장면 또는 주의 장면 / 표현 방법 또는 오늘 할 일 / 거리 조절 또는 돈 관리**
- 풀이 자세히 보기 / 올해 흐름 근거 보기 접힘 메뉴

**문제**:
- "올해 핵심" 문장이 모든 분야에서 9장 리포트와 동일 ("배움과 도움을 크게 쓰는 편이라…")
- "주의 장면"에 "현재는 丙申 긴 흐름과 丙午 올해 흐름이 함께 작동하므로…" — 한자 노출 + 명리 시점 용어
- 분야 간 본문 중복

**개선 방향**: 같은 사주 데이터를 **분야별 렌즈로** 풀어내야 함. "정인격(돌봄·후원의 별)"이라는 사실을:
- 직장 카드: "선배·멘토의 도움이 결정적인 시기"
- 재물 카드: "큰돈보다 인덕·교육 투자가 자산이 되는 시기"
- 연애 카드: "안정적으로 돌봐주는 인연이 어울리는 시기"
- 관계 카드: "받은 만큼 베푸는 관계 균형이 핵심"

각각 다른 문장으로 풀어야 합니다.

## 5. 콘텐츠 중복 맵

문제 진단 차원에서, 사이트 전체에서 같은 문장이 어디어디 등장하는지 정리합니다. 작업 우선순위 결정용 자료.

| 문장 (요약) | 명리 출처 | 등장 위치 | 등장 횟수 |
|------------|---------|----------|---------|
| "배움과 도움을 크게 쓰는 편이라 도움을 받고…" | 정인격 | 상세 1·4·5·6·9장 + 세운 직장·재물·연애·관계 카드 | **~8** |
| "표현은 충분히 살아 있고 기준은 비기 쉬워…" | 상관 + 인성 부족 | 상세 4·5·9장 + 세운 재물·관계 카드 | **~5** |
| "기준이 약해질 때는 만족보다 피로가 남는 소비가…" | 재성 약함 | 상세 5·9장 + 세운 재물 카드 | ~3 |
| "오늘은 새 투자보다 고정비, 미뤄둔 정산…" | 재성 운영 | 상세 5·9장 + 세운 재물 카드 | ~3 |
| "오늘은 '네가 항상' 같은 단정 대신…" | 인성·관성 | 상세 4·9장 + 세운 관계 카드 | ~3 |
| "기준이 흔들리면 같은 고민을 오래 붙잡기 쉽습니다." | 인성 과다 | 상세 2·9장 | 2 |
| "현재는 丙申 긴 흐름과 丙午 올해 흐름이…" | 대운+세운 | 상세 6장 + 세운 직장운 주의 장면 | 2 |
| "X 대운에는 천간의 [형용어]과 지지의 [형용어]과 결이…" | 대운 천간/지지 | 대운 9개 카드 (변수만 교체) | 9 |
| "X 대운의 결은 관계의 거리감과 표현 방식을…" | 대운 공통 | 대운 9개 카드 (관계 섹션) | 9 |
| "X 대운은 돈과 일의 결을 함께 재편합니다…" | 대운 공통 | 대운 9개 카드 (돈커리어 섹션) | 9 |

**총 추정**: 사이트의 본문 분량 중 **30~40%가 동일 문장의 다른 위치 등장**으로 추정됩니다.

## 8. 권장 작업 순서

가장 적은 공수로 가장 큰 체감 변화를 얻는 순서:

**Week 1**: P0 버그 6개 모두 수정 → 사이트의 즉각적 인상 변화

**Week 2~3**: "X과" 네이밍을 자연 비유(옵션 A)로 일괄 교체 → 톤 전체 부드러워짐

**Week 4**: 본문 한자 제거 + 자극적 표현 제거 + 자동 검증 시스템 가동 → 접근성 한 단계

**Month 2**: 대운 9개를 진짜 다르게 만드는 LLM 파이프라인 (`daewoon-llm-spec.md` 적용) → 대운 탭 콘텐츠가 살아남

**Month 3**: 9장 상세 리포트를 챕터별 다른 렌즈로 재생성 → 가장 큰 가치 작업. "같은 얘기 또 함" 문제 해소

**Month 4~6**: 분야운 카드·세운·일진 차례로 적용 + 품질 모니터링 + 사용자 피드백 루프
