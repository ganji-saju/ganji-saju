# 사주 풀이 용어 친근화 — 변경 기록 (2026-05-14)

> 작업 PR: feature/saju-terminology-simplify-2026-05-14
> 단일 소스: `src/lib/saju/terminology.ts`
> 적용 범위: v2 interpretation builder · v2 insight UI panel · `simplifySajuCopy` · 검증 메시지

## 1. 배경
v2 wrapper 가 들어오면서 풀이문이 다시 한자 술어 (`일간`, `용신`, `대운`, `세운`, `십신` …) 로 회귀했다는 사용자 피드백이 있었다.
요즘 세대가 빠르게 이해하기 어려운 용어들을 전수 검사해, 일상어로 일관되게 치환했다.

## 2. 원칙
1. **단정형 금지** — "반드시 / 무조건 / 100% / 확실히 일어" 같은 말은 빌더 단계에서 차단.
2. **한자 술어 1차 차단** — UI/풀이 본문에는 한자 술어가 노출되지 않도록 `FRIENDLY_TERM_MAP` 으로 자동 치환.
3. **검증·로그는 병기** — 개발자가 추적해야 할 검증 메시지(`issue.message`)에서는 쉬운 말이 1차, 한자 술어는 보조 괄호로.
4. **단일 소스** — `src/lib/saju/terminology.ts` 하나에서 매핑을 관리.
5. **부드러운 단어 우선** — "타고난 / 흐름 / 균형 / 기운 / 결" 같은 단어를 우선 사용.

## 5. 본문 카피 변경 (Block 별)

### 5.1 foundation (내 타고난 결)
- **이전**: "`{stem}` 일간은 `{metaphor}`의 이미지로 설명할 수 있습니다."
- **변경**: "내 핵심 기질은 `{나무/불/땅/쇠/물 기운}`의 결로 `{metaphor}`처럼 풀이됩니다."

### 5.2 five-elements-balance (다섯 기운 균형)
- **이전 summary**: "`{한자}({오행})` 점수가 가장 높고 `{한자}({오행})` 점수가 가장 낮습니다."
- **변경 summary**: "다섯 기운 중에서 `{나무 기운}`이 가장 강하고 `{쇠 기운}`이 가장 약하게 잡혀요. 약한 기운은 '부족함'이라기보다 '일상에서 작게 채워두면 좋은 결'로 봐주세요."
- **claim 변경**: 한자 오행 → 친근 라벨 + 한 줄 키워드 (FRIENDLY_ELEMENT_HINT) 보강.

### 5.3 yongsin (잘 풀리게 도와주는 기운)
- **이전**: "현재 1순위 용신은 `{label}`이며, 판정 방식은 `{method}`입니다."
- **변경**: "지금 가장 도움이 되는 핵심 기운은 `{나무 기운/불 기운/...}` (확신도 꽤 확실)이에요. 이 기운을 일상에서 작게 자주 써주면 균형이 부드러워져요."

### 5.4 luck-flow (지금 흐르는 운)
- **이전**: "현재 대운은 `{ganzi}`, 세운은 `{ganzi}`, 월운은 `{ganzi}`입니다."
- **변경**: "지금은 `{대운 ganzi}` 긴 흐름 위에 올라타 있어요. 올해 흐름은 `{세운 ganzi}`, 이번 달은 `{월운 ganzi}`으로 잡혀요."

### 5.5 executiveSummary
- **이전**: "일간은 `{stem 한자(오행)}` 입니다. 오행은 `{한자(오행)}` 우세, `{한자(오행)}` 약세로 읽힙니다. 강약 판정은 `{level} N점`, 용신 후보는 `{label} · 신뢰도 {높음}` 입니다."
- **변경**: "내 핵심 기질은 `{나무 기운의 metaphor}` 입니다. 다섯 기운 중 `{나무 기운}`이 강하고 `{쇠 기운}`이 약한 편이에요. 컨디션은 `{균형이 잡힌 편 (점수 70)}`, 도움이 되는 기운은 `{불 기운 · 확신도 꽤 확실}`으로 읽힙니다. 풀이 점검 점수는 N점 (`{문제 없음/작은 경고 있음/문제 있음 — 다시 계산 권장}`)."
